# `openclaw memory`

セマンティックメモリのインデックス化と検索を管理します。
アクティブなメモリプラグインによって提供されます（デフォルト: `memory-core`。`plugins.slots.memory = "none"`を設定して無効化）。

関連:
- Memoryコンセプト: [Memory](/concepts/memory)
- プラグイン: [Plugins](/plugins)

## 例

```bash
openclaw memory status
openclaw memory status --deep
openclaw memory status --deep --index
openclaw memory status --deep --index --verbose
openclaw memory index
openclaw memory index --verbose
openclaw memory search "release checklist"
openclaw memory status --agent main
openclaw memory index --agent main --verbose
```

## オプション

共通:

- `--agent <id>`: 単一のエージェントにスコープします（デフォルト: すべての設定されたエージェント）。
- `--verbose`: プローブとインデックス化中に詳細ログを出力します。

注意事項:
- `memory status --deep`は、vector + embeddingの可用性をプローブします。
- `memory status --deep --index`は、ストアがdirtyの場合に再インデックス化を実行します。
- `memory index --verbose`は、フェーズごとの詳細を表示します（プロバイダー、モデル、ソース、バッチアクティビティ）。
- `memory status`には、`memorySearch.extraPaths`経由で設定された追加のパスが含まれます。
