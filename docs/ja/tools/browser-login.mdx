# ブラウザログイン + X/Twitter投稿

## 手動ログイン（推奨）

サイトがログインを必要とする場合、**ホスト** ブラウザプロファイル（openclawブラウザ）で **手動でサインイン** してください。

モデルに認証情報を渡さ **ない** でください。自動ログインはアンチボット防御をトリガーすることが多く、アカウントがロックされる可能性があります。

メインのブラウザドキュメントに戻る: [ブラウザ](/tools/browser)。

## どのChromeプロファイルが使用されますか？

OpenClawは **専用のChromeプロファイル**（`openclaw` という名前、オレンジ色のUI）を制御します。これは日常のブラウザプロファイルとは別です。

アクセスする2つの簡単な方法:

1) **エージェントにブラウザを開くよう依頼** してから、自分でログインします。
2) **CLI経由で開く**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

複数のプロファイルがある場合は、`--browser-profile <name>` を渡します（デフォルトは `openclaw`）。

## X/Twitter: 推奨フロー

- **読み取り/検索/スレッド:** **bird** CLI スキル（ブラウザなし、安定）を使用します。
  - リポジトリ: https://github.com/steipete/bird
- **更新の投稿:** **ホスト** ブラウザ（手動ログイン）を使用します。

## サンドボックス化 + ホストブラウザアクセス

サンドボックス化されたブラウザセッションは、ボット検出をトリガーする **可能性が高く** なります。X/Twitter（および他の厳格なサイト）には、**ホスト** ブラウザを優先してください。

エージェントがサンドボックス化されている場合、ブラウザツールはデフォルトでサンドボックスになります。ホスト制御を許可するには:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true
        }
      }
    }
  }
}
```

次にホストブラウザをターゲットにします:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

または、更新を投稿するエージェントのサンドボックスを無効にします。
