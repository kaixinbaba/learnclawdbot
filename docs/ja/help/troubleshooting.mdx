# トラブルシューティング

## 最初の60秒

以下を順番に実行してください：

```bash
openclaw status
openclaw status --all
openclaw gateway probe
openclaw logs --follow
openclaw doctor
```

Gateway に到達可能な場合は、詳細な調査を実行：

```bash
openclaw status --deep
```

## よくある「壊れた」ケース

### `openclaw: command not found`

ほぼ必ず Node/npm の PATH の問題です。ここから始めてください：

- [インストール（Node/npm PATH の健全性チェック）](/install#nodejs--npm-path-sanity)

### インストーラの失敗（または詳細ログが必要な場合）

詳細なトレースと npm 出力を表示するため、verbose モードでインストーラを再実行してください：

```bash
curl -fsSL https://openclaw.bot/install.sh | bash -s -- --verbose
```

beta インストールの場合：

```bash
curl -fsSL https://openclaw.bot/install.sh | bash -s -- --beta --verbose
```

フラグの代わりに `OPENCLAW_VERBOSE=1` を設定することもできます。

- [Gateway トラブルシューティング](/gateway/troubleshooting)
- [Gateway 認証](/gateway/authentication)

### Control UI が HTTP で失敗する（デバイス ID が必要）

- [Gateway トラブルシューティング](/gateway/troubleshooting)

### `docs.openclaw.ai` で SSL エラーが表示される（Comcast/Xfinity）

一部の Comcast/Xfinity 接続では、Xfinity Advanced Security により `docs.openclaw.ai` がブロックされます。
Advanced Security を無効にするか、`docs.openclaw.ai` を許可リストに追加してから、再試行してください。

- Xfinity Advanced Security ヘルプ：https://www.xfinity.com/support/articles/using-xfinity-xfi-advanced-security
- クイック健全性チェック：モバイルホットスポットまたは VPN を試して、ISP レベルのフィルタリングであることを確認してください

### `/model` が `model not allowed` と表示する

これは通常、`agents.defaults.models` が許可リストとして設定されていることを意味します。空でない場合、
これらの provider/model キーのみが選択できます。

- 許可リストを確認：`openclaw config get agents.defaults.models`
- 必要なモデルを追加（または許可リストをクリア）して再試行 `/model`
- `/models` を使用して、許可された providers/models を参照してください

### Issue を提出する際

安全なレポートを貼り付けてください：

```bash
openclaw status --all
```

可能であれば、`openclaw logs --follow` から関連するログの末尾を含めてください。
