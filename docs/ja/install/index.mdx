# インストール

理由がない限りインストーラーを使用してください。CLIをセットアップし、オンボーディングを実行します。

## クイックインストール（推奨）

```bash
curl -fsSL https://openclaw.bot/install.sh | bash
```

Windows（PowerShell）:

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

次のステップ（オンボーディングをスキップした場合）:

```bash
openclaw onboard --install-daemon
```

## システム要件

- **Node >=22**
- macOS、Linux、またはWSL2経由のWindows
- ソースからビルドする場合のみ`pnpm`

## インストール方法の選択

### 1) インストーラースクリプト（推奨）

npm経由で`openclaw`をグローバルにインストールし、オンボーディングを実行します。

```bash
curl -fsSL https://openclaw.bot/install.sh | bash
```

インストーラーフラグ:

```bash
curl -fsSL https://openclaw.bot/install.sh | bash -s -- --help
```

詳細: [Installer internals](/install/installer)。

非対話型（オンボーディングをスキップ）:

```bash
curl -fsSL https://openclaw.bot/install.sh | bash -s -- --no-onboard
```

### 2) グローバルインストール（手動）

既にNodeがある場合:

```bash
npm install -g openclaw@latest
```

libvipsがグローバルにインストールされている場合（Homebrew経由のmacOSで一般的）で`sharp`のインストールに失敗する場合は、プリビルドバイナリを強制します:

```bash
SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm install -g openclaw@latest
```

`sharp: Please add node-gyp to your dependencies`が表示される場合は、ビルドツールをインストールするか（macOS: Xcode CLT + `npm install -g node-gyp`）、上記の`SHARP_IGNORE_GLOBAL_LIBVIPS=1`回避策を使用してネイティブビルドをスキップしてください。

または:

```bash
pnpm add -g openclaw@latest
```

次に:

```bash
openclaw onboard --install-daemon
```

### 3) ソースから（コントリビューター/開発者）

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build # 初回実行時にUI depsを自動インストール
pnpm build
openclaw onboard --install-daemon
```

ヒント: グローバルインストールがまだない場合は、`pnpm openclaw ...`経由でリポジトリコマンドを実行してください。

### 4) その他のインストールオプション

- Docker: [Docker](/install/docker)
- Nix: [Nix](/install/nix)
- Ansible: [Ansible](/install/ansible)
- Bun（CLIのみ）: [Bun](/install/bun)

## インストール後

- オンボーディングを実行: `openclaw onboard --install-daemon`
- クイックチェック: `openclaw doctor`
- Gatewayヘルスチェック: `openclaw status` + `openclaw health`
- ダッシュボードを開く: `openclaw dashboard`

## インストール方法: npm vs git（インストーラー）

インストーラーは2つの方法をサポートしています:

- `npm`（デフォルト）: `npm install -g openclaw@latest`
- `git`: GitHubからクローン/ビルドし、ソースチェックアウトから実行

### CLIフラグ

```bash
# 明示的なnpm
curl -fsSL https://openclaw.bot/install.sh | bash -s -- --install-method npm

# GitHubからインストール（ソースチェックアウト）
curl -fsSL https://openclaw.bot/install.sh | bash -s -- --install-method git
```

共通フラグ:

- `--install-method npm|git`
- `--git-dir <path>`（デフォルト: `~/openclaw`）
- `--no-git-update`（既存のチェックアウト使用時に`git pull`をスキップ）
- `--no-prompt`（プロンプトを無効化、CI/自動化で必須）
- `--dry-run`（何が起こるか表示、変更なし）
- `--no-onboard`（オンボーディングをスキップ）

### 環境変数

同等の環境変数（自動化に便利）:

- `OPENCLAW_INSTALL_METHOD=git|npm`
- `OPENCLAW_GIT_DIR=...`
- `OPENCLAW_GIT_UPDATE=0|1`
- `OPENCLAW_NO_PROMPT=1`
- `OPENCLAW_DRY_RUN=1`
- `OPENCLAW_NO_ONBOARD=1`
- `SHARP_IGNORE_GLOBAL_LIBVIPS=0|1`（デフォルト: `1`、システムlibvipsに対する`sharp`ビルドを回避）

## トラブルシューティング: `openclaw`が見つからない（PATH）

クイック診断:

```bash
node -v
npm -v
npm prefix -g
echo "$PATH"
```

`$(npm prefix -g)/bin`（macOS/Linux）または`$(npm prefix -g)`（Windows）が`echo "$PATH"`内に**存在しない**場合、シェルはnpmグローバルバイナリ（`openclaw`を含む）を見つけられません。

修正: シェル起動ファイル（zsh: `~/.zshrc`、bash: `~/.bashrc`）に追加してください:

```bash
# macOS / Linux
export PATH="$(npm prefix -g)/bin:$PATH"
```

Windowsでは、`npm prefix -g`の出力をPATHに追加してください。

次に新しいターミナルを開いてください（またはzshで`rehash` / bashで`hash -r`）。
