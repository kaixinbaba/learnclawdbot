# モデル設定の探索

このドキュメントは、将来のモデル設定のアイデアを記録しています。これは出荷仕様ではありません。現在の動作については、以下を参照してください：

- [モデル](/concepts/models)
- [モデルフェイルオーバー](/concepts/model-failover)
- [OAuth + プロファイル](/concepts/oauth)

## 動機

オペレーターは以下を求めています：

- プロバイダーごとに複数の認証プロファイル（個人 vs 仕事）
- 予測可能なフォールバックを持つシンプルな `/model` 選択
- テキストモデルと画像対応モデルの明確な分離

## 可能な方向性（高レベル）

- モデル選択をシンプルに保つ：provider/model にオプションのエイリアスを追加
- プロバイダーが複数の認証プロファイルを持ち、明示的な順序を持つようにする
- すべてのセッションが一貫してフェイルオーバーするよう、グローバルフォールバックリストを使用
- 明示的に設定された場合にのみ画像ルーティングをオーバーライド
