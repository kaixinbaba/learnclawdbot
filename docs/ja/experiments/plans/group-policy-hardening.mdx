# Telegram 許可リスト堅牢化

**日付：** 2026-01-05  
**ステータス：** 完了  
**PR：** #216

## 概要

Telegram 許可リストは、大文字小文字を区別せずに `telegram:` および `tg:` プレフィックスを受け入れ、偶発的な空白を許容するようになりました。これにより、インバウンド許可リストチェックがアウトバウンド送信正規化と一致します。

## 変更内容

- プレフィックス `telegram:` と `tg:` は同じものとして扱われます（大文字小文字を区別しない）
- 許可リストエントリはトリミングされます；空のエントリは無視されます

## 例

同じ ID に対して、以下のすべてが受け入れられます：

- `telegram:123456`
- `TG:123456`
- `tg:123456`

## なぜ重要か

ログやチャット ID からのコピー/ペーストには、プレフィックスや空白が含まれることがよくあります。正規化により、DM またはグループで応答するかどうかを決定する際の誤検知を回避できます。
