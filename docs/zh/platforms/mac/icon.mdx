# 菜单栏图标状态

作者：steipete · 更新时间：2025-12-06 · 范围：macOS 应用（`apps/macos`）

- **空闲（Idle）：** 正常图标动画（眨眼、偶尔摆动）。
- **暂停（Paused）：** 状态项使用 `appearsDisabled`；无运动。
- **语音触发（大耳朵）：** 语音唤醒检测器在听到唤醒词时调用 `AppState.triggerVoiceEars(ttl: nil)`，在捕获话语期间保持 `earBoostActive=true`。耳朵放大（1.9倍），获得圆形耳孔以提高可读性，然后在 1 秒静默后通过 `stopVoiceEars()` 降下。仅从应用内语音管道触发。
- **工作中（agent 运行中）：** `AppState.isWorking=true` 驱动"尾巴/腿部跑动"微动作：工作进行时更快的腿部摆动和轻微偏移。当前在 WebChat agent 运行周围切换；当你连接其他长任务时添加相同的切换。

连接点
- 语音唤醒：runtime/tester 在触发时调用 `AppState.triggerVoiceEars(ttl: nil)`，在 1 秒静默后调用 `stopVoiceEars()` 以匹配捕获窗口。
- Agent 活动：在工作跨度周围设置 `AppStateStore.shared.setWorking(true/false)`（已在 WebChat agent 调用中完成）。保持跨度简短，并在 `defer` 块中重置以避免卡住的动画。

形状和尺寸
- 基础图标在 `CritterIconRenderer.makeIcon(blink:legWiggle:earWiggle:earScale:earHoles:)` 中绘制。
- 耳朵缩放默认为 `1.0`；语音增强设置 `earScale=1.9` 并切换 `earHoles=true`，不改变整体帧（18×18 pt 模板图像渲染到 36×36 px Retina 背景存储）。
- 跑动使用腿部摆动最多约 1.0，带有小的水平抖动；它与任何现有的空闲摆动叠加。

行为注意事项
- 耳朵/工作没有外部 CLI/broker 切换；保持应用自身信号的内部性以避免意外抖动。
- 保持 TTL 简短（<10 秒），以便在作业挂起时图标快速返回基线。
