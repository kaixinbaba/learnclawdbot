# Scripts (脚本)

`scripts/` 目录包含用于本地工作流和运维任务的辅助脚本。
当任务明确与脚本相关时使用这些;否则优先使用 CLI。

## 约定

- 脚本是**可选的**,除非在文档或发布检查清单中被引用。
- 当存在 CLI 表面时优先使用(示例: 认证监控使用 `openclaw models status --check`)。
- 假设脚本是特定于主机的;在新机器上运行前先阅读它们。

## Git hooks (钩子)

- `scripts/setup-git-hooks.js`: 在 git 仓库内时为 `core.hooksPath` 进行尽力而为的设置。
- `scripts/format-staged.js`: 暂存的 `src/` 和 `test/` 文件的 pre-commit 格式化器。
