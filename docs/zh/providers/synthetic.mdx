# Synthetic

Synthetic 暴露 Anthropic 兼容的端点。OpenClaw 将其注册为
`synthetic` provider 并使用 Anthropic Messages API。

## 快速设置

1) 设置 `SYNTHETIC_API_KEY`(或运行下面的向导)。
2) 运行 onboarding:

```bash
openclaw onboard --auth-choice synthetic-api-key
```

默认模型设置为:

```
synthetic/hf:MiniMaxAI/MiniMax-M2.1
```

## 配置示例

```json5
{
  env: { SYNTHETIC_API_KEY: "sk-..." },
  agents: {
    defaults: {
      model: { primary: "synthetic/hf:MiniMaxAI/MiniMax-M2.1" },
      models: { "synthetic/hf:MiniMaxAI/MiniMax-M2.1": { alias: "MiniMax M2.1" } }
    }
  },
  models: {
    mode: "merge",
    providers: {
      synthetic: {
        baseUrl: "https://api.synthetic.new/anthropic",
        apiKey: "${SYNTHETIC_API_KEY}",
        api: "anthropic-messages",
        models: [
          {
            id: "hf:MiniMaxAI/MiniMax-M2.1",
            name: "MiniMax M2.1",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 192000,
            maxTokens: 65536
          }
        ]
      }
    }
  }
}
```

注意: OpenClaw 的 Anthropic 客户端会在 base URL 后附加 `/v1`,所以使用
`https://api.synthetic.new/anthropic`(不是 `/anthropic/v1`)。如果 Synthetic 更改了
base URL,覆盖 `models.providers.synthetic.baseUrl`。

## 注意事项

- 模型引用使用 `synthetic/<modelId>`。
- 如果你启用了模型 allowlist (允许列表)(`agents.defaults.models`),添加你
  计划使用的每个模型。
- 查看 [Model providers](/concepts/model-providers) 了解 provider 规则。
