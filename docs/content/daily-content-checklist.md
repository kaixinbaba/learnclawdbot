## 📅 每日工作流程

### 1️⃣ 早晨（规划）

```markdown
□ 查看今日计划（参考 60 天计划表）
□ 确认内容类型和目标关键词
□ 准备所需素材（图片、数据、案例）
□ 设置写作目标和截止时间
```

### 2️⃣ 上午（创作）

```markdown
□ 按照模板开始写作
□ 完成初稿（不追求完美）
□ 插入标题和段落结构
□ 达到目标字数要求
```

### 3️⃣ 下午（优化）

```markdown
□ SEO 优化
  □ 检查关键词密度
  □ 优化 title 和 meta description
  □ 添加内部链接
  □ 插入 Alt text

□ 内容完善
  □ 添加图片和截图
  □ 检查语法和拼写
  □ 优化可读性
  □ 添加 CTA 按钮
```

### 4️⃣ 傍晚（发布）

```markdown
□ 最终检查
  □ 预览移动端显示
  □ 测试所有链接
  □ 验证 Schema markup
  □ 检查加载速度

□ 发布流程
  □ 发布到网站
  □ 提交到 Google Search Console
  □ 分享到社交媒体
  □ 记录发布日志
```

## 📝 内容类型快速指南

### 核心页面（1000-1500 词）

```yaml
结构:
  - Introduction (100 words)
  - Core Content (600-800 words)
  - Benefits (200 words)
  - FAQ (200-300 words)
  - CTA (100 words)

关键元素:
  - 5-8 个 H2 标题
  - 3-5 张图片
  - 1 个 Schema markup
  - 5-7 个内链

估计时间: 3-4 小时
```

### 功能页面（1200-1500 词）

```yaml
结构:
  - Problem Statement (150 words)
  - Solution Overview (200 words)
  - Step-by-Step Guide (400 words)
  - Benefits (200 words)
  - Examples (250 words)
  - FAQ (200 words)
  - CTA (100 words)

关键元素:
  - 6-8 个 H2/H3 标题
  - 4-6 张截图
  - HowTo Schema
  - 6-8 个内链

估计时间: 4-5 小时
```

### 博客教程（1200-1800 词）

```yaml
结构:
  - Hook Introduction (150 words)
  - Table of Contents
  - Main Content - 3 sections (300 words each)
  - Examples (200 words)
  - Key Takeaways (150 words)
  - Conclusion + CTA (100 words)

关键元素:
  - 5-7 个 H2 标题
  - 5-8 张图片/截图
  - Article Schema
  - 8-10 个内链

估计时间: 4-6 小时
```

### 简短博客（800-1200 词）

```yaml
结构:
  - Introduction (100 words)
  - Main Points (500-800 words)
  - Conclusion (100 words)
  - CTA (100 words)

关键元素:
  - 3-5 个 H2 标题
  - 3-4 张图片
  - Article Schema
  - 5-6 个内链

估计时间: 2-3 小时
```

## 🎯 内容创作模板

### 快速启动模板

#### 标题创作（5 分钟）

```markdown
使用这些公式快速生成标题：

1. 数字公式: "[数字] Ways clawdbot [动词] [结果]"
   示例: "5 Ways clawdbot Saves You 2 Hours Daily"

2. How-to 公式: "How to [动词] [关键词] Without [痛点]"
   示例: "How to Manage Email Without Opening Your Inbox"

3. 对比公式: "[产品] vs [竞品]: Which [比较维度]"
   示例: "clawdbot vs ChatGPT: Which Handles Tasks Better"

4. 问题公式: "[疑问词] [关键词] [具体问题]"
   示例: "Can You Really Manage Email from WhatsApp?"
```

#### 开头段落模板（10 分钟）

```markdown
第一段（痛点 + 数据）:
"[描述痛点]. [统计数据显示问题严重性]."

第二段（解决方案）:
"clawdbot [如何解决]. By [核心特点], you can [具体好处]."

第三段（文章价值）:
"In this guide, you'll learn:
• [要点 1]
• [要点 2]
• [要点 3]"

第四段（可信度 - 可选）:
"Over [数字] [用户类型] already use clawdbot to [结果]."
```

#### 主体内容模板

```markdown
## Section 1: [H2 标题]

[引言句子介绍这部分内容] (50 words)

### [H3 副标题]
[详细解释] (150-200 words)

[插入图片/截图]

### [H3 副标题]
[详细解释] (150-200 words)

**Key takeaway:** [总结要点]

---

## Section 2: [H2 标题]

[同上结构重复]

---

## Section 3: [H2 标题]

[同上结构重复]
```

#### 结尾模板（5 分钟）

```markdown
## Key Takeaways

Here's what we covered:

• [要点 1]
• [要点 2]
• [要点 3]
• [要点 4]
• [要点 5]

## Ready to Get Started?

[一句话总结文章价值]. [行动号召].

[CTA 按钮: "Try clawdbot Free" / "Learn More" / "Get Started"]

## Related Articles

- [相关文章 1 标题](链接)
- [相关文章 2 标题](链接)
- [相关文章 3 标题](链接)
```

## ✅ 发布前检查清单

### SEO 元素（5 分钟）

```markdown
□ Title tag
  □ 包含目标关键词
  □ 55-60 字符长度
  □ 吸引点击的标题

□ Meta description
  □ 包含目标关键词
  □ 150-160 字符
  □ 包含 CTA

□ URL
  □ 简短易读
  □ 包含关键词
  □ 使用连字符分隔

□ Headings
  □ 一个 H1（与 title 类似但不完全相同）
  □ 5-8 个 H2 标题
  □ H3 标题用于细分

□ Images
  □ 所有图片都有 Alt text
  □ 文件名描述性强
  □ 图片已优化（< 100KB）
  □ 使用 WebP 格式
```

### 内容质量（10 分钟）

```markdown
□ 结构
  □ 引言吸引人
  □ 逻辑流畅
  □ 段落简短（2-3 句）
  □ 使用列表和要点

□ 可读性
  □ 无语法错误
  □ 无拼写错误
  □ 句子长度适中
  □ 避免行话（或解释行话）

□ SEO
  □ 目标关键词在前 100 词出现
  □ 关键词密度合理（1-2%）
  □ 包含 LSI 关键词
  □ 自然不刻意

□ 价值
  □ 提供实用信息
  □ 有具体例子
  □ 可执行的建议
  □ 解决用户问题
```

### 链接（5 分钟）

```markdown
□ Internal Links (内链)
  □ 5-7 个相关内链
  □ 锚文本描述性强
  □ 链接到相关内容
  □ 无 broken links

□ External Links (外链)
  □ 1-2 个权威来源
  □ 在新窗口打开
  □ 使用 rel="noopener"
  □ 链接高质量网站
```

### 多媒体（10 分钟）

```markdown
□ Images
  □ 至少 3 张相关图片
  □ 高质量清晰
  □ 统一风格
  □ 移动端友好

□ Screenshots
  □ 清晰易读
  □ 标注重点
  □ 适当裁剪
  □ 隐藏敏感信息

□ Videos (可选)
  □ 嵌入 YouTube/Vimeo
  □ 添加字幕
  □ 设置适当尺寸
```

### 技术实现（10 分钟）

```markdown
□ Schema Markup
  □ 选择正确类型（Article/HowTo/FAQ）
  □ 无语法错误
  □ 通过 Rich Results Test

□ Open Graph
  □ og:title
  □ og:description
  □ og:image (1200x630px)
  □ og:url

□ Mobile
  □ 移动端预览正常
  □ 按钮可点击
  □ 文字可读
  □ 加载速度快

□ Performance
  □ 页面加载 < 3 秒
  □ 图片懒加载
  □ 无 console 错误
```
