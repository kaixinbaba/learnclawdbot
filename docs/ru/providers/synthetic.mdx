# Synthetic

Synthetic предоставляет конечные точки, совместимые с Anthropic. OpenClaw регистрирует его как
провайдер `synthetic` и использует Anthropic Messages API.

## Быстрая настройка

1) Установите `SYNTHETIC_API_KEY` (или запустите мастер ниже).
2) Запустите процесс настройки:

```bash
openclaw onboard --auth-choice synthetic-api-key
```

Модель по умолчанию установлена на:

```
synthetic/hf:MiniMaxAI/MiniMax-M2.1
```

## Пример конфигурации

```json5
{
  env: { SYNTHETIC_API_KEY: "sk-..." },
  agents: {
    defaults: {
      model: { primary: "synthetic/hf:MiniMaxAI/MiniMax-M2.1" },
      models: { "synthetic/hf:MiniMaxAI/MiniMax-M2.1": { alias: "MiniMax M2.1" } }
    }
  },
  models: {
    mode: "merge",
    providers: {
      synthetic: {
        baseUrl: "https://api.synthetic.new/anthropic",
        apiKey: "${SYNTHETIC_API_KEY}",
        api: "anthropic-messages",
        models: [
          {
            id: "hf:MiniMaxAI/MiniMax-M2.1",
            name: "MiniMax M2.1",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 192000,
            maxTokens: 65536
          }
        ]
      }
    }
  }
}
```

Примечание: клиент Anthropic в OpenClaw добавляет `/v1` к базовому URL, поэтому используйте
`https://api.synthetic.new/anthropic` (не `/anthropic/v1`). Если Synthetic изменит
свой базовый URL, переопределите `models.providers.synthetic.baseUrl`.

## Примечания

- Ссылки на модели используют `synthetic/<modelId>`.
- Если вы включите список разрешенных моделей (`agents.defaults.models`), добавьте каждую модель, которую
  планируете использовать.
- См. [Провайдеры моделей](/concepts/model-providers) для правил провайдера.
