# Исследование конфигурации моделей

Этот документ фиксирует идеи для будущей конфигурации моделей. Это не спецификация для выпуска. Для текущего поведения см.:

- [Модели](/concepts/models)
- [Переключение модели при сбое](/concepts/model-failover)
- [OAuth + профили](/concepts/oauth)

## Мотивация

Операторы хотят:

- Несколько профилей аутентификации на провайдера (личный vs рабочий)
- Простой выбор `/model` с предсказуемыми резервами
- Четкое разделение между текстовыми моделями и моделями с поддержкой изображений

## Возможное направление (высокий уровень)

- Сохранить простой выбор модели: provider/model с опциональными псевдонимами
- Позволить провайдерам иметь несколько профилей аутентификации с явным порядком
- Использовать глобальный список резервов, чтобы все сессии переключались единообразно
- Переопределять маршрутизацию изображений только при явной конфигурации

## Открытые вопросы

- Должна ли ротация профилей быть для каждого провайдера или для каждой модели?
- Как UI должен отображать выбор профиля для сессии?
- Каков самый безопасный путь миграции от устаревших ключей конфигурации?
