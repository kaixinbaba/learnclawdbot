# Отслеживание использования

## Что это
- Получает данные об использовании/квотах провайдера напрямую с их конечных точек использования.
- Без оценочных затрат; только окна, о которых сообщает провайдер.

## Где это отображается
- `/status` в чатах: карточка статуса с эмодзи с токенами сессии + оценочной стоимостью (только API ключ). Использование провайдера отображается для **текущего провайдера модели**, когда доступно.
- `/usage off|tokens|full` в чатах: футер использования для каждого ответа (OAuth показывает только токены).
- `/usage cost` в чатах: локальная сводка стоимости, агрегированная из логов сессий OpenClaw.
- CLI: `openclaw status --usage` выводит полную разбивку по провайдерам.
- CLI: `openclaw channels list` выводит тот же снимок использования вместе с конфигурацией провайдера (используйте `--no-usage` для пропуска).
- Строка меню macOS: раздел "Usage" в Context (только если доступно).

## Провайдеры + учетные данные
- **Anthropic (Claude)**: токены OAuth в профилях аутентификации.
- **GitHub Copilot**: токены OAuth в профилях аутентификации.
- **Gemini CLI**: токены OAuth в профилях аутентификации.
- **Antigravity**: токены OAuth в профилях аутентификации.
- **OpenAI Codex**: токены OAuth в профилях аутентификации (accountId используется при наличии).
- **MiniMax**: API ключ (ключ плана кодирования; `MINIMAX_CODE_PLAN_KEY` или `MINIMAX_API_KEY`); использует окно 5-часового плана кодирования.
- **z.ai**: API ключ через env/config/хранилище аутентификации.

Использование скрыто, если соответствующие учетные данные OAuth/API не существуют.
