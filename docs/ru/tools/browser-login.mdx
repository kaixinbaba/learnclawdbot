# Вход в браузер + публикация в X/Twitter

## Ручной вход (рекомендуется)

Когда сайт требует входа, **войдите вручную** в профиле браузера **хоста** (браузер openclaw).

**Не** давайте модели свои учетные данные. Автоматические входы часто вызывают защиту от ботов и могут заблокировать аккаунт.

Вернуться к основной документации по браузеру: [Браузер](/tools/browser).

## Какой профиль Chrome используется?

OpenClaw управляет **выделенным профилем Chrome** (с именем `openclaw`, пользовательский интерфейс с оранжевым оттенком). Это отдельно от вашего ежедневного профиля браузера.

Два простых способа получить к нему доступ:

1) **Попросите агента открыть браузер**, а затем войдите самостоятельно.
2) **Откройте через CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

Если у вас несколько профилей, передайте `--browser-profile <имя>` (по умолчанию `openclaw`).

## X/Twitter: рекомендуемый подход

- **Чтение/поиск/треды:** используйте навык CLI **bird** (без браузера, стабильно).
  - Репозиторий: https://github.com/steipete/bird
- **Публикация обновлений:** используйте браузер **хоста** (ручной вход).

## Изолированная среда + доступ к браузеру хоста

Сеансы браузера в изолированной среде **с большей вероятностью** вызывают обнаружение ботов. Для X/Twitter (и других строгих сайтов) предпочтительнее браузер **хоста**.

Если агент находится в изолированной среде, инструмент браузера по умолчанию использует изолированную среду. Чтобы разрешить управление хостом:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true
        }
      }
    }
  }
}
```

Затем нацельтесь на браузер хоста:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

Или отключите изоляцию для агента, который публикует обновления.
