# `openclaw models`

Обнаружение моделей, сканирование и конфигурация (модель по умолчанию, откаты, профили авторизации).

Связанные разделы:
- Провайдеры + модели: [Models](/providers/models)
- Настройка авторизации провайдера: [Getting started](/start/getting-started)

## Основные команды

```bash
openclaw models status
openclaw models list
openclaw models set <model-or-alias>
openclaw models scan
```

`openclaw models status` показывает разрешённые по умолчанию/откаты плюс обзор авторизации.
Когда доступны снимки использования провайдера, раздел статуса OAuth/токена включает
заголовки использования провайдера.
Добавьте `--probe` для запуска живых проверок авторизации для каждого настроенного профиля провайдера.
Проверки — это реальные запросы (могут потреблять токены и вызывать ограничения скорости).

Примечания:
- `models set <model-or-alias>` принимает `provider/model` или псевдоним.
- Ссылки на модели парсятся разделением по **первому** `/`. Если ID модели включает `/` (стиль OpenRouter), включите префикс провайдера (пример: `openrouter/moonshotai/kimi-k2`).
- Если вы опустите провайдера, OpenClaw обрабатывает ввод как псевдоним или модель для **провайдера по умолчанию** (работает только когда нет `/` в ID модели).

### `models status`
Опции:
- `--json`
- `--plain`
- `--check` (выход 1=истёк/отсутствует, 2=истекает)
- `--probe` (живая проверка настроенных профилей авторизации)
- `--probe-provider <name>` (проверка одного провайдера)
- `--probe-profile <id>` (повторить или id профилей разделённые запятыми)
- `--probe-timeout <ms>`
- `--probe-concurrency <n>`
- `--probe-max-tokens <n>`

## Псевдонимы + откаты

```bash
openclaw models aliases list
openclaw models fallbacks list
```

## Профили авторизации

```bash
openclaw models auth add
openclaw models auth login --provider <id>
openclaw models auth setup-token
openclaw models auth paste-token
```
`models auth login` запускает поток авторизации плагина провайдера (OAuth/API ключ). Используйте
`openclaw plugins list` чтобы увидеть какие провайдеры установлены.

Примечания:
- `setup-token` запрашивает значение setup-token (сгенерируйте его с помощью `claude setup-token` на любой машине).
- `paste-token` принимает строку токена сгенерированную в другом месте или из автоматизации.
