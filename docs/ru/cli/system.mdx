# `openclaw system`

Системные помощники для Gateway: постановка в очередь системных событий, управление heartbeat-опросами и просмотр присутствия.

## Основные команды

```bash
openclaw system event --text "Check for urgent follow-ups" --mode now
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## `system event`

Постановка в очередь системного события в **основной** сессии. Следующий heartbeat-опрос вставит его как строку `System:` в промпт. Используйте `--mode now` для немедленного запуска heartbeat-опроса; `next-heartbeat` ожидает следующего запланированного такта.

Флаги:
- `--text <text>`: обязательный текст системного события.
- `--mode <mode>`: `now` или `next-heartbeat` (по умолчанию).
- `--json`: машиночитаемый вывод.

## `system heartbeat last|enable|disable`

Управление heartbeat-опросами:
- `last`: показать последнее событие heartbeat.
- `enable`: снова включить heartbeat-опросы (используйте, если они были отключены).
- `disable`: приостановить heartbeat-опросы.

Флаги:
- `--json`: машиночитаемый вывод.

## `system presence`

Список текущих записей системного присутствия, о которых знает Gateway (узлы, экземпляры и аналогичные строки статуса).

Флаги:
- `--json`: машиночитаемый вывод.

## Примечания

- Требуется запущенный Gateway, доступный по вашей текущей конфигурации (локальный или удаленный).
- Системные события эфемерны и не сохраняются при перезапуске.
