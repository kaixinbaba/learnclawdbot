# Устранение неполадок

## Первые 60 секунд

Выполните эти команды по порядку:

```bash
openclaw status
openclaw status --all
openclaw gateway probe
openclaw logs --follow
openclaw doctor
```

Если шлюз доступен, глубокая проверка:

```bash
openclaw status --deep
```

## Типичные случаи «что-то сломалось»

### `openclaw: command not found`

Почти всегда это проблема PATH для Node/npm. Начните здесь:

- [Установка (проверка PATH для Node/npm)](/install#nodejs--npm-path-sanity)

### Установщик завершается с ошибкой (или нужны полные логи)

Перезапустите установщик в подробном режиме, чтобы увидеть полную трассировку и вывод npm:

```bash
curl -fsSL https://openclaw.bot/install.sh | bash -s -- --verbose
```

Для бета-установки:

```bash
curl -fsSL https://openclaw.bot/install.sh | bash -s -- --beta --verbose
```

Вы также можете установить `OPENCLAW_VERBOSE=1` вместо флага.

### `docs.openclaw.ai` показывает ошибку SSL (Comcast/Xfinity)

Некоторые подключения Comcast/Xfinity блокируют `docs.openclaw.ai` через Xfinity Advanced Security.
Отключите Advanced Security или добавьте `docs.openclaw.ai` в список разрешённых, затем повторите попытку.

- Справка Xfinity Advanced Security: https://www.xfinity.com/support/articles/using-xfinity-xfi-advanced-security
- Быстрая проверка: попробуйте мобильную точку доступа или VPN, чтобы подтвердить, что это фильтрация на уровне провайдера

### `/model` говорит `model not allowed`

Обычно это означает, что `agents.defaults.models` настроен как список разрешённых. Когда он не пустой,
могут быть выбраны только эти ключи провайдера/модели.

- Проверьте список разрешённых: `openclaw config get agents.defaults.models`
- Добавьте нужную модель (или очистите список разрешённых) и повторите `/model`
- Используйте `/models` для просмотра разрешённых провайдеров/моделей

### При создании issue

Вставьте безопасный отчёт:

```bash
openclaw status --all
```

Если возможно, включите хвост логов из `openclaw logs --follow`.
