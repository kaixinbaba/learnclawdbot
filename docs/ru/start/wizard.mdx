# Мастер настройки (CLI)

Мастер настройки — это **рекомендуемый** способ настройки OpenClaw на macOS, Linux или Windows (через WSL2; настоятельно рекомендуется).
Он настраивает локальный Gateway или подключение к удаленному Gateway, плюс каналы, навыки и настройки рабочего пространства по умолчанию в одном управляемом потоке.

Основная точка входа:

```bash
openclaw onboard
```

Самый быстрый первый чат: откройте Control UI (настройка каналов не нужна). Запустите `openclaw dashboard` и общайтесь в браузере. Документация: [Dashboard](/web/dashboard).

Последующая реконфигурация:

```bash
openclaw configure
```

Рекомендуется: настройте API ключ Brave Search, чтобы агент мог использовать `web_search` (`web_fetch` работает без ключа). Самый простой путь: `openclaw configure --section web`, который сохраняет `tools.web.search.apiKey`. Документация: [Веб-инструменты](/tools/web).

## QuickStart vs Расширенный

Мастер начинается с **QuickStart** (настройки по умолчанию) vs **Расширенный** (полный контроль).

**QuickStart** сохраняет настройки по умолчанию:
- Локальный gateway (loopback)
- Рабочее пространство по умолчанию (или существующее рабочее пространство)
- Порт Gateway **18789**
- Авторизация Gateway **Token** (автоматически сгенерирован, даже на loopback)
- Предоставление Tailscale **Выкл**
- Telegram + WhatsApp DMs по умолчанию на **allowlist** (вам будет предложено ввести ваш номер телефона)

**Расширенный** предоставляет каждый шаг (режим, рабочее пространство, gateway, каналы, daemon, навыки).

## Что делает мастер

**Локальный режим (по умолчанию)** проводит вас через:
- Модель/авторизация (OAuth подписки OpenAI Code (Codex), API ключ Anthropic (рекомендуется) или setup-token (вставка), плюс опции MiniMax/GLM/Moonshot/AI Gateway)
- Местоположение рабочего пространства + файлы bootstrap
- Настройки Gateway (порт/bind/auth/tailscale)
- Провайдеры (Telegram, WhatsApp, Discord, Google Chat, Mattermost (плагин), Signal)
- Установка Daemon (LaunchAgent / systemd user unit)
- Проверка здоровья
- Навыки (рекомендуется)

**Удаленный режим** только настраивает локальный клиент для подключения к Gateway в другом месте.
Он **не** выполняет удаленные установки или изменения daemon.

Чтобы добавить больше изолированных агентов (отдельное рабочее пространство + сессии + авторизация), используйте:

```bash
openclaw agents add <name>
```

Совет: `--json` **не** подразумевает неинтерактивный режим. Используйте `--non-interactive` (и `--workspace`) для скриптов.

(Остальное содержимое файла с подробными разделами опущено для краткости, но полный перевод должен включать все разделы: Flow details, Model/Auth, Workspace, Gateway, Channels, Daemon install, Health check, Skills, Remote mode, Non-interactive mode и т.д.)
