# Скрипты

Каталог `scripts/` содержит вспомогательные скрипты для локальных рабочих процессов и операционных задач.
Используйте их, когда задача четко привязана к скрипту; в остальных случаях предпочитайте CLI.

## Соглашения

- Скрипты **опциональны**, если не упомянуты в документации или чеклистах релизов.
- Предпочитайте интерфейсы CLI, когда они существуют (пример: мониторинг аутентификации использует `openclaw models status --check`).
- Предполагайте, что скрипты специфичны для хоста; читайте их перед запуском на новой машине.

## Git hooks

- `scripts/setup-git-hooks.js`: настройка с разумными усилиями для `core.hooksPath` внутри git-репозитория.
- `scripts/format-staged.js`: форматировщик перед коммитом для staged файлов `src/` и `test/`.

## При добавлении скриптов

- Держите скрипты сфокусированными и задокументированными.
- Добавьте короткую запись в соответствующей документации (или создайте её, если отсутствует).
