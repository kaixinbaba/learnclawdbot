# VPS 호스팅

이 허브는 지원되는 VPS/호스팅 가이드로 연결되며, 클라우드 배포가 높은 수준에서 어떻게 작동하는지 설명합니다.

## Provider 선택

- **Railway** (원클릭 + 브라우저 설정): [Railway](/railway)
- **Northflank** (원클릭 + 브라우저 설정): [Northflank](/northflank)
- **Oracle Cloud (Always Free)**: [Oracle](/platforms/oracle) — $0/월 (Always Free, ARM; 용량/가입이 까다로울 수 있음)
- **Fly.io**: [Fly.io](/platforms/fly)
- **Hetzner (Docker)**: [Hetzner](/platforms/hetzner)
- **GCP (Compute Engine)**: [GCP](/platforms/gcp)
- **exe.dev** (VM + HTTPS 프록시): [exe.dev](/platforms/exe-dev)
- **AWS (EC2/Lightsail/free tier)**: 잘 작동합니다. 비디오 가이드:
  https://x.com/techfrenAJ/status/2014934471095812547

## 클라우드 설정 작동 방식

- **Gateway는 VPS에서 실행**되며 상태 + workspace를 소유합니다.
- **Control UI** 또는 **Tailscale/SSH**를 통해 노트북/휴대폰에서 연결합니다.
- VPS를 신뢰의 소스로 취급하고 상태 + workspace를 **백업**하세요.
- 보안 기본값: Gateway를 loopback에 유지하고 SSH 터널 또는 Tailscale Serve를 통해 액세스합니다.
  `lan`/`tailnet`에 바인딩하는 경우, `gateway.auth.token` 또는 `gateway.auth.password`가 필요합니다.

원격 액세스: [Gateway remote](/gateway/remote)  
플랫폼 허브: [Platforms](/platforms)

## VPS와 함께 node 사용

Gateway를 클라우드에 유지하면서 로컬 장치(Mac/iOS/Android/헤드리스)에 **node**를 페어링할 수 있습니다. Node는 Gateway가 클라우드에 유지되는 동안 로컬 화면/카메라/canvas 및 `system.run` 기능을 제공합니다.

문서: [Nodes](/nodes), [Nodes CLI](/cli/nodes)
