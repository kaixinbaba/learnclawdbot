# `openclaw models`

Model 검색, 스캔 및 구성 (기본 model, fallback, auth profile).

관련 문서:
- Provider + model: [Models](/providers/models)
- Provider 인증 설정: [Getting started](/start/getting-started)

## 공통 명령

```bash
openclaw models status
openclaw models list
openclaw models set <model-or-alias>
openclaw models scan
```

`openclaw models status`는 해결된 기본/fallback과 인증 개요를 표시합니다.
provider 사용량 스냅샷을 사용할 수 있는 경우, OAuth/token 상태 섹션에 
provider 사용량 헤더가 포함됩니다.
각 구성된 provider profile에 대해 라이브 인증 프로브를 실행하려면 `--probe`를 추가합니다.
프로브는 실제 요청입니다 (토큰을 소비하고 rate limit을 트리거할 수 있음).

참고:
- `models set <model-or-alias>`는 `provider/model` 또는 alias를 허용합니다.
- Model ref는 **첫 번째** `/`로 분할하여 파싱됩니다. model ID에 `/`가 포함된 경우(OpenRouter 스타일), provider 접두사를 포함하세요 (예: `openrouter/moonshotai/kimi-k2`).
- provider를 생략하면 OpenClaw는 입력을 alias 또는 **기본 provider**의 model로 처리합니다 (model ID에 `/`가 없는 경우에만 작동).

### `models status`
옵션:
- `--json`
- `--plain`
- `--check` (exit 1=만료/누락, 2=만료 예정)
- `--probe` (구성된 auth profile의 라이브 프로브)
- `--probe-provider <name>` (하나의 provider 프로브)
- `--probe-profile <id>` (반복 또는 쉼표로 구분된 profile id)
- `--probe-timeout <ms>`
- `--probe-concurrency <n>`
- `--probe-max-tokens <n>`

## Alias + fallback

```bash
openclaw models aliases list
openclaw models fallbacks list
```

## Auth profile

```bash
openclaw models auth add
openclaw models auth login --provider <id>
openclaw models auth setup-token
openclaw models auth paste-token
```
`models auth login`은 provider plugin의 인증 플로우(OAuth/API key)를 실행합니다.
설치된 provider를 확인하려면 `openclaw plugins list`를 사용하세요.

참고:
- `setup-token`은 setup-token 값을 요청합니다 (모든 머신에서 `claude setup-token`으로 생성).
- `paste-token`은 다른 곳에서 생성된 token 문자열 또는 자동화에서 생성된 token을 허용합니다.
