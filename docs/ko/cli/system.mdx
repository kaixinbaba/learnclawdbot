# `openclaw system`

Gateway를 위한 시스템 레벨 헬퍼: system event 큐 추가, heartbeat 제어,
presence 보기.

## 공통 명령

```bash
openclaw system event --text "Check for urgent follow-ups" --mode now
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## `system event`

**main** session에 system event를 큐에 추가합니다. 다음 heartbeat가 
프롬프트에 `System:` 라인으로 주입합니다. `--mode now`를 사용하면 heartbeat를 
즉시 트리거합니다; `next-heartbeat`는 다음 예약된 틱을 기다립니다.

플래그:
- `--text <text>`: 필수 system event 텍스트.
- `--mode <mode>`: `now` 또는 `next-heartbeat` (기본값).
- `--json`: 머신 읽기 가능한 출력.

## `system heartbeat last|enable|disable`

Heartbeat 제어:
- `last`: 마지막 heartbeat event 표시.
- `enable`: heartbeat 다시 켜기 (비활성화된 경우 사용).
- `disable`: heartbeat 일시 중지.

플래그:
- `--json`: 머신 읽기 가능한 출력.

## `system presence`

Gateway가 알고 있는 현재 system presence 항목을 나열합니다 (node,
인스턴스 및 유사한 상태 라인).

플래그:
- `--json`: 머신 읽기 가능한 출력.

## 참고

- 현재 config(로컬 또는 원격)에서 도달 가능한 실행 중인 Gateway가 필요합니다.
- System event는 임시적이며 재시작 시 지속되지 않습니다.
