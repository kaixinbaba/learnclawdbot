# 채널 문제 해결

다음으로 시작하세요:

```bash
openclaw doctor
openclaw channels status --probe
```

`channels status --probe`는 일반적인 채널 구성 오류를 감지할 수 있을 때 경고를 출력하며,
작은 라이브 체크(자격 증명, 일부 권한/멤버십)를 포함합니다.

## 채널
- Discord: [/channels/discord#troubleshooting](/channels/discord#troubleshooting)
- Telegram: [/channels/telegram#troubleshooting](/channels/telegram#troubleshooting)

## Telegram 빠른 수정
- 로그에 `HttpError: Network request for 'sendMessage' failed` 또는 `sendChatAction`이 표시됨 → IPv6 DNS를 확인하세요. `api.telegram.org`가 IPv6로 먼저 확인되고 호스트에 IPv6 egress가 없으면 IPv4를 강제하거나 IPv6를 활성화하세요. [/channels/telegram#troubleshooting](/channels/telegram#troubleshooting)을 참조하세요.
- 로그에 `setMyCommands failed`가 표시됨 → `api.telegram.org`에 대한 아웃바운드 HTTPS 및 DNS 도달 가능성을 확인하세요 (잠긴 VPS 또는 프록시에서 일반적).
