# 개념

OpenClaw를 구동하는 기본 개념과 아키텍처를 이해합니다.

## 핵심 개념

- **[에이전트](agent)** - OpenClaw 에이전트란 무엇인가
- **[에이전트 루프](agent-loop)** - 에이전트가 메시지를 처리하고 작업을 실행하는 방법
- **[에이전트 워크스페이스](agent-workspace)** - 에이전트의 파일 시스템과 메모리
- **[아키텍처](architecture)** - 시스템 아키텍처와 구성 요소
- **[채널 라우팅](channel-routing)** - 채널 간 메시지 라우팅 방법
- **[세션](sessions)** - 대화 세션 및 컨텍스트 관리
- **[사고 및 추론](thinking-reasoning)** - 에이전트 의사결정 프로세스
- **[마크다운 포맷팅](markdown-formatting)** - 크로스 플랫폼 메시지 포맷팅

## 고급 주제

- **[서브에이전트](subagents)** - 작업을 위한 전문 서브에이전트 생성
- **[도구 사용](tool-use)** - 에이전트가 도구를 발견하고 사용하는 방법
- **[A2UI](a2ui)** - 에이전트-UI 통신 프로토콜

OpenClaw의 작동 방식을 깊이 이해하려면 위의 주제를 살펴보세요.
