# 모델 설정 탐색

이 문서는 미래 모델 설정에 대한 아이디어를 포착합니다. 이것은 출시 사양이 아닙니다. 현재 동작은 다음을 참조하세요:

- [모델](/concepts/models)
- [모델 페일오버](/concepts/model-failover)
- [OAuth + 프로필](/concepts/oauth)

## 동기

운영자가 원하는 것:

- 프로바이더당 여러 인증 프로필 (개인 vs 직장)
- 예측 가능한 폴백이 있는 간단한 `/model` 선택
- 텍스트 모델과 이미지 지원 모델 간의 명확한 분리

## 가능한 방향 (상위 수준)

- 모델 선택을 간단하게 유지: 선택적 별칭이 있는 provider/model
- 프로바이더가 명시적 순서를 가진 여러 인증 프로필을 갖도록 함
- 모든 세션이 일관되게 페일오버하도록 글로벌 폴백 목록 사용
- 명시적으로 설정된 경우에만 이미지 라우팅 재정의

## 미해결 질문

- 프로필 로테이션은 프로바이더별로 해야 하는가, 모델별로 해야 하는가?
- UI는 세션에 대한 프로필 선택을 어떻게 표시해야 하는가?
- 레거시 설정 키로부터 가장 안전한 마이그레이션 경로는 무엇인가?
