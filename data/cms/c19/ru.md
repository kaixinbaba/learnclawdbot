# Офлайн-ИИ: запуск OpenClaw с Ollama для полной конфиденциальности

В тот момент, когда вы отправляете сообщение любому облачному ИИ-сервису, это сообщение покидает ваш компьютер. Оно проходит через CDN, записывается в логи провайдера и может быть использовано для обучения моделей или проверено людьми. Для большинства повседневных задач это приемлемый компромисс. Но для всего остального — проприетарного кода, персональных данных, конфиденциальной бизнес-логики, секретных документов — это неприемлемо.

Я начал исследовать локальный ИИ по конкретной причине: мне нужно было использовать ИИ-ассистента для анализа тикетов службы поддержки, содержащих персональные данные (PII). Использование облачного API для этого создало бы реальные проблемы с соответствием нормативным требованиям. Обработка всего локально решила проблему чисто и элегантно. С тех пор я значительно расширил локальную инфраструктуру и теперь использую её как основную среду для всего, что требует конфиденциальности.

Это руководство охватывает запуск OpenClaw полностью в офлайн-режиме с использованием Ollama в качестве бэкенда для моделей. По завершении у вас будет ИИ-ассистент, который работает без подключения к интернету, обрабатывает все данные локально и не требует никаких API-ключей.

## Почему Ollama

Ollama — самый простой способ запустить большие языковые модели локально. Он берёт на себя загрузку моделей, конвертацию в формат GGUF, определение аппаратного ускорения и предоставляет REST API, совместимый со спецификацией OpenAI Chat Completions. Последний пункт как раз и позволяет работать с OpenClaw без написания специального кода интеграции.

Существуют и другие варианты — LM Studio, прямой запуск llama.cpp, LocalAI, — но Ollama находит оптимальный баланс между простотой и функциональностью. Вы устанавливаете его, скачиваете модель — и он поднимает API. OpenClaw подключается к этому API точно так же, как подключается к OpenAI.

## Системные требования

Локальные модели требуют реальных вычислительных ресурсов. Реалистичный минимум по сценариям использования:

**Базовое использование (модели с 7B параметров):**
- RAM: 8 ГБ (модель частично работает в RAM при отсутствии GPU)
- VRAM GPU: 4 ГБ для аппаратного ускорения
- Хранилище: 5–10 ГБ на модель

**Комфортная производительность (модели 13B):**
- RAM: 16 ГБ
- VRAM GPU: 8 ГБ
- Хранилище: 8–15 ГБ на модель

**Высокое качество (модели 30B и выше):**
- RAM: 32 ГБ и более
- VRAM GPU: 16 ГБ и более (или Apple Silicon с унифицированной памятью)
- Хранилище: 20–50 ГБ на модель

Mac на Apple Silicon (M1/M2/M3/M4) имеют унифицированную память и особенно хорошо справляются с локальными моделями. MacBook Pro с 32 ГБ унифицированной памяти запускает модели 34B без затруднений.

## Установка Ollama

```bash
# macOS / Linux
curl -fsSL https://ollama.com/install.sh | sh

# Windows: скачайте установщик с ollama.com

# Проверка установки
ollama --version
```

Запустите сервис Ollama:

```bash
ollama serve
# На macOS после установки запускается как приложение в строке меню
```

## Выбор модели

Выбор модели — самое важное решение в этой конфигурации. Больше не всегда лучше: небольшая модель, которая помещается в VRAM GPU, будет значительно быстрее крупной модели, работающей в оперативной памяти.

**Лучшие модели для программирования и технических задач:**

```bash
# Специализируется на коде, отлично для технических вопросов
ollama pull codellama:13b

# Qwen2.5-Coder: превосходное понимание кода
ollama pull qwen2.5-coder:14b

# DeepSeek-Coder: силён в автодополнении кода
ollama pull deepseek-coder-v2:16b
```

**Лучшие модели для написания текстов и анализа:**

```bash
# Llama 3.1: лучшая универсальная модель в этом размере
ollama pull llama3.1:8b

# Mistral: быстрая, хорошее качество при 7B
ollama pull mistral:7b

# Gemma 2: открытая модель от Google, хорошие рассуждения
ollama pull gemma2:9b
```

**Крупные модели для максимального качества:**

```bash
# Llama 3.1 70B (требует ~40 ГБ RAM или 24+ ГБ VRAM)
ollama pull llama3.1:70b

# Qwen2.5 32B (лучше многих более крупных моделей на отдельных задачах)
ollama pull qwen2.5:32b
```

Протестируйте модель перед подключением OpenClaw:

```bash
ollama run mistral:7b "Explain what an MCP server is in one paragraph"
```

Если это работает и качество ответа приемлемо, вы готовы к подключению.

## Настройка OpenClaw для работы с Ollama

OpenClaw подключается к Ollama через его OpenAI-совместимый API. Отредактируйте конфигурацию OpenClaw:

```yaml
# ~/.openclaw/config.yaml

providers:
  - name: ollama-local
    type: openai-compatible
    baseUrl: "http://localhost:11434/v1"
    apiKey: "ollama"  # Ollama не требует настоящего ключа, но поле должно присутствовать
    models:
      - id: mistral:7b
        name: "Mistral 7B (Local)"
        contextWindow: 32768
      - id: llama3.1:8b
        name: "Llama 3.1 8B (Local)"
        contextWindow: 131072
      - id: codellama:13b
        name: "CodeLlama 13B (Local)"
        contextWindow: 16384

defaultProvider: ollama-local
defaultModel: mistral:7b
```

Перезапустите OpenClaw и убедитесь, что подключение работает:

```bash
openclaw status
# Должно показать: Provider: ollama-local, Model: mistral:7b
```

## Проверка офлайн-работы

После настройки убедитесь, что всё работает без доступа к интернету:

```bash
# Отключить сеть (macOS)
networksetup -setnetworkserviceenabled Wi-Fi off

# Протестировать OpenClaw
openclaw chat "What is 15 * 37?"

# Включить сеть обратно
networksetup -setnetworkserviceenabled Wi-Fi on
```

Ответ должен прийти даже при отключённой сети. Если OpenClaw зависает или выдаёт ошибку, проверьте, что Ollama по-прежнему работает (он должен работать — после загрузки модели Ollama не требует интернета).

## Оптимизация производительности

Локальные модели медленнее облачных API. Вот что реально влияет на скорость:

**Аппаратное ускорение GPU — это самый важный фактор.** Проверьте, использует ли Ollama ваш GPU:

```bash
ollama run mistral:7b "" --verbose
# Ищите: "using GPU: NVIDIA GeForce..." или "using Metal: Apple M..."
```

Если при наличии совместимого GPU запуск идёт только на CPU, установите соответствующие драйверы (CUDA для NVIDIA, ROCm для AMD). На macOS с Apple Silicon ускорение через Metal активируется автоматически.

**Количество параллельных потоков при работе на CPU:**

```bash
# В переменных окружения Ollama (на Linux создайте /etc/ollama/ollama.conf)
OLLAMA_NUM_PARALLEL=4
OLLAMA_MAX_LOADED_MODELS=1
```

**Размер контекста существенно влияет на скорость.** Окно контекста 128K токенов на модели 7B работает медленнее, чем контекст 8K. Если длинный контекст не нужен, настройте его явно:

```yaml
# В конфигурации OpenClaw
providers:
  - name: ollama-local
    models:
      - id: mistral:7b
        contextWindow: 8192  # Меньший контекст = более быстрые ответы
```

**Держите модель загруженной.** Ollama выгружает модели после таймаута. В процессе разработки держите её в горячем состоянии:

```bash
# Поддерживать модель загруженной, отправляя пустой запрос каждые несколько минут
while true; do
  curl -s http://localhost:11434/api/generate \
    -d '{"model":"mistral:7b","prompt":"","keep_alive":"10m"}' > /dev/null
  sleep 300
done
```

## Практические сценарии использования

**Код-ревью проприетарной кодовой базы:**

```bash
cd /path/to/private-project
openclaw chat "Review this file for security issues" --files src/auth/jwt.ts
```

Ничего не покидает ваш компьютер. Весь анализ кода выполняется локально.

**Анализ конфиденциальных документов:**

```bash
openclaw chat "Summarize the key obligations in this contract" --files contract.pdf
```

Работает с PDF, текстовыми файлами, файлами кода — любым форматом, поддерживаемым обработчиком файлов OpenClaw.

**Работа в изолированной среде без интернета:**

Некоторые производственные среды не имеют доступа к интернету. После того как Ollama и OpenClaw установлены и модели загружены, весь стек работает в air-gapped окружении.

**Поиск по локальной базе знаний:**

Сочетайте с загрузкой контекста в OpenClaw:

```bash
openclaw chat --context /path/to/internal/docs "How does our payment integration work?"
```

Модель отвечает, используя только вашу локальную документацию.

## Настройка нескольких моделей

Разные задачи выигрывают от разных моделей. OpenClaw поддерживает переключение моделей в процессе разговора:

```yaml
# ~/.openclaw/config.yaml
providers:
  - name: ollama-local
    type: openai-compatible
    baseUrl: "http://localhost:11434/v1"
    apiKey: "ollama"
    models:
      - id: mistral:7b
        name: "Fast (Mistral 7B)"
      - id: codellama:13b
        name: "Code (CodeLlama 13B)"
      - id: llama3.1:70b
        name: "Quality (Llama 3.1 70B)"
```

На практике я использую Mistral 7B для быстрых вопросов и итераций, CodeLlama для технического анализа, а Llama 70B — для сложных задач рассуждения, где меня не смущает более долгое ожидание.

## Сравнение качества: облако против локальных моделей

Честная оценка: лучшие локальные модели (70B и выше) близки к лучшим облачным моделям (GPT-4o, Claude 3.5 Sonnet) для сложных задач, но не равны им. Разрыв значительно сократился в 2025 году. Для большинства практических задач:

- **Помощь с кодом**: локальные модели 13B и выше очень хороши. Незначительное преимущество облака при самом сложном рефакторинге.
- **Суммирование текста**: локальные модели отлично справляются. Минимальная разница в качестве начиная с 8B.
- **Творческое письмо**: облачные модели по-прежнему впереди, но локальная 70B вполне пригодна.
- **Рассуждения и анализ**: облачные модели лучше для нестандартных сложных задач. Локальные вполне достаточны для устоявшихся паттернов.

Для работы с конфиденциальными данными такой компромисс в качестве обычно того стоит. Для максимального качества при работе с нечувствительными данными облачные API по-прежнему остаются лучшим выбором.

## Часто задаваемые вопросы

**Можно ли использовать модели Ollama с функциями вызова инструментов и MCP в OpenClaw?**

Большинство моделей Ollama поддерживают вызов функций / использование инструментов. Совместимость зависит от модели. CodeLlama, Llama 3.1 и Mistral поддерживают это хорошо. Проверьте документацию модели на сайте Ollama. При использовании MCP-серверов с локальными моделями протестируйте корректность вызова инструментов моделью — некоторые небольшие модели испытывают трудности со сложными схемами инструментов.

**Как обновлять модели Ollama?**

```bash
ollama pull mistral:7b  # Повторный pull загружает последнюю версию
ollama list             # Посмотреть все загруженные модели
ollama rm mistral:7b    # Удалить модель для освобождения места
```

**Можно ли запускать несколько моделей одновременно?**

Да, но каждая загруженная модель занимает память. На системе с 32 ГБ обычно можно запустить две модели 7B одновременно. Задайте `OLLAMA_MAX_LOADED_MODELS=2` в переменных окружения Ollama.

**Работает ли сохранение памяти и сессий OpenClaw с локальными моделями?**

Да. Управление сессиями в OpenClaw не зависит от бэкенда модели. Локальные сессии, загрузка контекста и история разговоров одинаково работают с Ollama.

**О квантовании — что лучше выбрать, Q4 или Q8?**

Ollama по умолчанию использует квантование Q4_K_M, которое обеспечивает хороший баланс. Q8 даёт более высокое качество за счёт большего потребления памяти. Для модели 7B: Q4 использует ~4 ГБ VRAM, Q8 — ~8 ГБ. Если VRAM позволяет, Q8 того стоит. Квантование задаётся при загрузке:

```bash
ollama pull mistral:7b-instruct-q8_0
```

Качество достаточно высокое, чтобы я редко ощущал необходимость переключаться на облачные API для задач, требующих конфиденциальности. А гарантия приватности абсолютна: никакие данные никогда не покидают машину.
