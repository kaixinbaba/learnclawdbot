# 10 个提升生产力的必备 OpenClaw 插件

开箱即用的 OpenClaw 已经是一个能干的 AI 助手。但它真正的威力来自**插件**——这些模块化扩展将你的助手连接到你实际使用的工具。插件不只是包装器；它赋予 AI 真正行动的能力，而不只是给出建议。

## OpenClaw 插件的工作原理

每个插件由一个清单文件（`openclaw.plugin.json`）定义，描述该插件能做什么。清单告诉 OpenClaw：

- 插件暴露的**操作**（如 `search_web`、`create_issue`、`send_message`）
- 所需的**认证信息**（API Key、OAuth Token）
- AI 用于判断何时调用它的自然语言**描述**

运行时，当你说"帮我找一下 X 的最新资讯"，AI 会看到谷歌搜索插件的描述并决定使用它。你不需要显式调用插件——它们在相关时自动激活。

你可以在 [ClawdHub](https://clawdhub.com) 上浏览并安装社区插件。每个列表页展示插件的操作、所需凭证、用户评分和更新日志。

## 10 大必备插件

### 1. Google Search（谷歌搜索）

**功能**：执行实时网络搜索，将结构化结果、新闻或页面摘要直接带入你的对话上下文。

**为什么重要**：没有这个插件，你的 AI 助手的知识就停在训练截止日期。有了它，你可以询问时事、最新软件版本、实时价格，或任何其他随时变化的信息。

**实际案例**："React 19 引入了哪些破坏性变更？"——助手搜索、阅读相关文档，给出具体答案，而不是从训练数据中猜测。

**在哪里找**：在 [ClawdHub](https://clawdhub.com) 搜索"Google Search"。需要 Google Custom Search API Key（低用量有免费额度）。

---

### 2. Browser Relay（浏览器中继）

**功能**：控制无界面浏览器抓取、渲染并提取任意网页的内容，包括依赖 JavaScript 渲染、会屏蔽简单爬虫的网站。

**为什么重要**：搜索引擎返回链接，这个插件给 AI 提供完整页面内容。对于竞争对手分析、研究，或阅读你有合法访问权的付费内容来说不可或缺。

**实际案例**："总结这篇研究论文 [URL] 的关键发现"——插件抓取完整页面，AI 阅读后给出结构化摘要。

**一个注意事项**：该插件在 JavaScript 渲染量大的页面上可能较慢。对于简单静态页面，谷歌搜索插件的摘要提取通常更快。

---

### 3. File System Manager（文件系统管理器）

**功能**：在本地机器或连接的硬盘上读取、写入、移动、复制和删除文件及目录。

**为什么重要**：这是 AI 辅助与你实际文件之间的桥梁。不再需要来回复制粘贴内容，AI 直接操作你的文件系统。

**实际案例**："找出 `/var/log` 目录下过去 24 小时内修改过的所有 `.log` 文件，汇总 ERROR 行，将摘要保存到桌面。"

**安全提示**：仔细配置插件的允许路径。不要授权访问整个文件系统——将其限制在它需要的目录。

---

### 4. GitHub Integration（GitHub 集成）

**功能**：跨 GitHub 仓库创建 Issue、提交 PR、读取文件内容、搜索代码、发布评论，以及检查工作流状态。

**为什么重要**：将 AI 助手变成真正能采取行动的编程协作者——不只是建议你该做什么。

**实际案例**：调试对话结束后，告诉它："为这个 bug 在 `auth` 仓库创建一个 GitHub Issue，包含我们刚才梳理的描述和复现步骤。"它会以规范格式创建 Issue。

**需要**：具备相应仓库权限的 GitHub 个人访问令牌（Personal Access Token）。

---

### 5. Twilio Voice（Twilio 语音）

**功能**：通过 Twilio 的 API 拨打和接听电话、发送短信，将电话与 AI 助手桥接。

**为什么重要**：实现真正的语音界面——从任意电话拨打你的 AI 助手。呼叫方无需安装任何应用。

**最佳用途**：这个插件是构建完整语音助手的基础——完整的配置教程（含 STT/TTS 设置）请参阅[语音助手配置指南](/blog/voice-assistant-openclaw)。

**需要**：一个带有电话号码的 Twilio 账号和凭证。Twilio 按通话/短信计费，价格见其官网。

---

### 6. Calendar（日历，支持 Google / Outlook）

**功能**：跨 Google Calendar 和 Microsoft Outlook 读取、创建、更新和删除日历事件，包括检查与会者的可用时间。

**为什么重要**：当 AI 助手能查看你的真实日历而不是盲目建议时间时，它在日程安排上才真正有用。

**实际案例**："明天上午留出两个小时用于深度工作，但先确认一下我没有已有安排。"它检查你的日历，找到空档，创建时间块。

**需要**：授权你的 Google 或 Microsoft 账户（OAuth 认证）。

---

### 7. Email（邮件，支持 Gmail / Outlook）

**功能**：从连接的账户读取、起草、发送、搜索和整理邮件。

**为什么重要**：邮件处理是投入产出比最高的可委托任务之一。"总结过去 24 小时的未读邮件，标记看起来紧急的内容"是一个真正有用的日常工作流。

**实际案例**："起草一封对上周二发给 X 公司提案的礼貌跟进邮件，控制在 100 字以内。"它找到原始发件邮件，读取上下文，然后起草跟进。

**需要**：OAuth 授权。插件不存储你的邮件凭证——它使用可随时撤销的 OAuth Token。

---

### 8. WhatsApp

**功能**：通过 WhatsApp Business API 或支持的第三方桥接发送和接收 WhatsApp 消息。

**为什么重要**：大量团队和家庭主要在 WhatsApp 上沟通。这个插件将你的 AI 引入这些频道——用于个人、家庭协调或小型商业客户互动。

**实际配置提示**：官方 WhatsApp Business API 需要企业认证。个人使用的话，部分第三方桥接提供更简便的接入——在 ClawdHub 查看当前可用选项和状态，因为这个集成生态变化较快。

---

### 9. Slack

**功能**：在 Slack 中发布消息、读取频道历史、搜索工作区内容、回复提及，并创建定时推送。

**为什么重要**：对于已经在 Slack 里工作的团队，这将 AI 引入现有工作流，而不是要求大家切换到另一个工具。

**实际案例**："总结昨天 #engineering 频道的讨论，并根据已解决的内容向 #team-updates 发布简短的状态更新。"它读取频道历史，发布摘要。

**需要**：具备相应 OAuth 权限的 Slack 应用（ClawdHub 上的插件页面会列出所需的权限范围）。

---

### 10. ClawdHub Marketplace（ClawdHub 市场）

**功能**：直接在 OpenClaw 的对话界面中浏览、安装、更新和删除插件，无需触碰任何配置文件。

**为什么重要**：插件生态在持续增长。Notion、Linear、Airtable、Jira 等工具的新集成插件不断出现在 ClawdHub 上，这个插件让跟进这些新增内容变得简单。

**实际案例**："ClawdHub 上有哪些 Notion 插件？"——它查询市场并展示当前可用选项及说明。

## 如何安装插件

**通过 Dashboard（推荐）**：

1. 打开 OpenClaw Dashboard，访问 `http://localhost:3000`
2. 进入 **插件 → 浏览**
3. 按名称搜索或按分类浏览
4. 点击 **安装**，提供所需的 API Key 或 OAuth 授权
5. 插件立即可用——让 OpenClaw 使用它来测试效果

**直接通过 ClawdHub**：

访问 [clawdhub.com](https://clawdhub.com)，找到插件后按其安装说明操作。每个插件页面在开始之前会明确列出所需凭证。

**手动安装**（适用于自定义或私有插件）：

将你的插件目录（包含 `openclaw.plugin.json` 和相关文件）放在 OpenClaw 的插件文件夹中，然后重启 OpenClaw。具体的插件目录路径请参阅 [docs.openclaw.ai](https://docs.openclaw.ai)。

## 常见问题

**插件安全吗？它们能访问哪些数据？**
每个插件只能访问你明确授权的内容。API Key 和 OAuth Token 存储在你的 OpenClaw 实例中，不会传输到 ClawdHub。在插件的 ClawdHub 页面可以查看其清单（manifest），了解它请求的确切权限。

**我能自己开发插件吗？**
可以。插件是一个包含 `openclaw.plugin.json` 清单和处理操作所需的服务端代码的文件夹。[OpenClaw 开发者文档](https://docs.openclaw.ai) 涵盖了清单格式和本地测试方法，然后再发布到 ClawdHub。

**插件兼容所有 AI 模型吗？**
兼容——插件与模型无关。AI 模型（DeepSeek、GPT-4o、Claude 等）读取插件清单，根据对话内容决定何时调用它。不同模型在调用插件的稳定性上可能略有差异，但所有支持的模型都与插件系统兼容。

**如果插件失效或停止工作怎么办？**
大多数故障是由 API 凭证过期或上游服务变更导致的。查看插件在 ClawdHub 上的页面了解已知问题和更新。你可以随时从 Dashboard 禁用插件而不必卸载它。
