# 10 обязательных плагинов OpenClaw для повышения продуктивности

Из коробки OpenClaw уже является полноценным ИИ-ассистентом. Но его настоящая сила раскрывается через **плагины** — модульные расширения, которые связывают ассистента с реальными инструментами, которыми вы пользуетесь каждый день. Веб-поиск, управление репозиториями, отправка сообщений в WhatsApp — для каждой задачи найдётся свой плагин.

## Что такое плагины OpenClaw?

Плагины OpenClaw определяются файлом манифеста `openclaw.plugin.json`. Каждый плагин объявляет:

- **Имя** и **описание** (используется ИИ для определения, когда следует вызывать плагин)
- **Действия**, которые он предоставляет (например, `search`, `create_issue`, `send_message`)
- Требования к **аутентификации** (API-ключи, OAuth-токены)
- Опциональные **триггеры** (вебхуки, расписания)

ИИ считывает эти манифесты во время работы и может вызывать любое доступное действие плагина, когда сочтёт его уместным для вашего запроса. Просматривать и устанавливать плагины сообщества можно на [ClawdHub](https://clawdhub.com).

## ТОП 10 плагинов

### 1. Google Search (Поиск Google)

**Что умеет**: Выполняет поиск в интернете в режиме реального времени и возвращает структурированные результаты, новости или сниппеты прямо в контекст чата.

**Зачем нужен**: Даёт ИИ-ассистенту доступ к актуальной информации — котировки акций, новости, документация, вышедшая позже даты обучения модели.

**Лучшее применение**: «Какая сейчас LTS-версия Node.js?» или «Найди свежие отзывы на [продукт]»

---

### 2. Browser Relay (Браузерный ретранслятор)

**Что умеет**: Управляет безголовым браузером для получения, рендеринга и извлечения содержимого любой веб-страницы, включая сайты с JavaScript-рендерингом, которые блокируют обычные scrapers.

**Зачем нужен**: Когда поиск возвращает ссылку, а вам нужно полное содержимое страницы.

**Лучшее применение**: «Перескажи статью по ссылке [URL]» или «Извлеки таблицу цен с [сайта конкурента]»

---

### 3. File System Manager (Менеджер файловой системы)

**Что умеет**: Читать, писать, перемещать, копировать и удалять файлы и директории на локальной машине или подключённых дисках.

**Зачем нужен**: Позволяет ИИ управлять реальными файлами — наводить порядок в загрузках, переименовывать фото пакетом, читать лог-файлы для диагностики проблем.

**Лучшее применение**: «Найди все файлы `.log` в `/var/log`, изменённые сегодня, и кратко опиши ошибки»

---

### 4. GitHub Integration (Интеграция с GitHub)

**Что умеет**: Взаимодействует с репозиториями GitHub — создаёт Issues, открывает Pull Requests, читает файлы, ищет по коду, публикует комментарии.

**Зачем нужен**: Превращает ИИ-ассистента в соавтора кода, который реально может делать коммиты и отслеживать задачи.

**Лучшее применение**: «Создай GitHub Issue для бага, который мы только что обсудили» или «Какие открытые PR ждут моего ревью?»

---

### 5. Twilio Voice (Голосовая связь Twilio)

**Что умеет**: Совершает и принимает звонки, отправляет SMS и соединяет телефонию с ИИ-ассистентом через API Twilio.

**Зачем нужен**: Открывает полноценный голосовой интерфейс — позвонить своему ИИ-ассистенту с любого телефона.

**Лучшее применение**: В связке с провайдером TTS/STT для создания полностью голосового ассистента (см. [туториал по голосовому ассистенту](/blog/voice-assistant-openclaw))

---

### 6. Calendar (Календарь, Google / Outlook)

**Что умеет**: Читает, создаёт, обновляет и удаляет события в Google Calendar и Microsoft Outlook.

**Зачем нужен**: ИИ-ассистент видит ваше расписание и помогает планировать — «Заблокируй 2 часа завтра утром для глубокой работы» или «Какие встречи у меня на этой неделе?»

**Лучшее применение**: «Запланируй 30-минутный звонок с Иваном на следующий вторник между 10:00 и 12:00»

---

### 7. Email (Почта, Gmail / Outlook)

**Что умеет**: Читает, составляет, отправляет и организует письма из подключённых аккаунтов.

**Зачем нужен**: Позволяет делегировать ИИ сортировку входящих и написание черновиков. «Кратко изложи непрочитанные письма за последние 24 часа» или «Напиши фолоу-ап по предложению, отправленному в прошлую пятницу».

**Лучшее применение**: В связке с Calendar для полной автоматизации цикла «входящие → календарь»

---

### 8. WhatsApp

**Что умеет**: Отправляет и получает сообщения WhatsApp через WhatsApp Business API или поддерживаемые сторонние мосты.

**Зачем нужен**: Многие команды и семьи общаются преимущественно в WhatsApp — включите своего ИИ в эти беседы.

**Лучшее применение**: Личный ИИ-ассистент, которому члены семьи могут задавать вопросы через WhatsApp без доступа к полному интерфейсу OpenClaw

---

### 9. Slack

**Что умеет**: Публикует сообщения, читает историю каналов, ищет по контенту воркспейса и отвечает на упоминания в Slack.

**Зачем нужен**: Встраивает ИИ-ассистента в командные рабочие процессы. «Опубликуй апдейт для стендапа в #engineering» или «Найди в Slack обсуждения аварии с API на прошлой неделе».

**Лучшее применение**: Автоматические ежедневные сводки по ключевым каналам в личные сообщения каждое утро

## Как установить плагины

Установка плагина в OpenClaw занимает три шага:

**Шаг 1: Просмотр ClawdHub**

Перейдите на [clawdhub.com](https://clawdhub.com) и найдите нужный плагин. На каждой странице указаны действия плагина, необходимые учётные данные и оценки сообщества.

**Шаг 2: Установка через Dashboard**

1. Откройте Dashboard OpenClaw по адресу `http://localhost:3000`
2. Перейдите в **Плагины → Обзор**
3. Найдите плагин по названию
4. Нажмите **Установить**
5. Введите необходимые API-ключи или авторизуйтесь через OAuth

**Шаг 3: Проверка**

После установки попросите OpenClaw воспользоваться плагином:

> «Используй плагин GitHub, чтобы показать список моих открытых Issues»

ИИ автоматически вызовет плагин и вернёт результаты.

**Ручная установка** (для кастомных или приватных плагинов):

```bash
# Поместите папку плагина в:
/home/user/openclaw/plugins/my-plugin/
# Убедитесь, что openclaw.plugin.json присутствует, затем перезапустите:
npm run restart
```
