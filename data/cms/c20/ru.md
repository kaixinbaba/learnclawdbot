# OpenClaw + Home Assistant: управление умным домом с помощью ИИ

Стандартный способ автоматизации умного дома — писать автоматизации: если солнце зашло и кто-то дома, включить свет в гостиной; если дверь открылась в 7 утра в будний день, запустить кофемашину. Всё это работает, но требует заранее предусмотреть сценарии и точно задать условия в коде.

Мне нужно было другое: возможность сказать «здесь немного жарко, сделай так, чтобы было комфортнее» — и чтобы ИИ сам разобрался, какие сущности изменить, насколько и в каком порядке. Это принципиально иная модель взаимодействия, и она работает на удивление хорошо.

В этом руководстве описывается подключение OpenClaw к Home Assistant через HA MCP сервер. После настройки вы сможете управлять домом с помощью разговорного языка, просить OpenClaw создавать автоматизации, диагностировать проблемы устройств и запрашивать состояние дома.

## Обзор архитектуры

Интеграция работает через три компонента:

1. **Home Assistant** — ваша существующая установка HA, любая версия от 2024 и выше
2. **Home Assistant MCP сервер** — мост, который открывает API Home Assistant в виде MCP-инструментов
3. **OpenClaw** — ИИ, который вызывает эти инструменты по вашим запросам

Когда вы говорите «выключи весь свет в спальне», OpenClaw отправляет это модели, модель решает вызвать инструмент `call_service` с `light.turn_off` и идентификаторами сущностей спальни, MCP сервер выполняет этот вызов к HA, и HA выключает свет.

Время полного цикла для простых команд — обычно 1–3 секунды от ввода до действия.

## Предварительные требования

- Установленный и запущенный Home Assistant (HA OS, Supervised, Container или Core)
- Долгосрочный токен доступа (Long-Lived Access Token) из HA
- Установленный OpenClaw
- Node.js версии 20 и выше

## Получение токена доступа Home Assistant

В Home Assistant:
1. Нажмите на свой профиль (левый нижний угол)
2. Прокрутите до раздела «Долгосрочные токены доступа» (Long-Lived Access Tokens)
3. Нажмите «Создать токен» (Create Token)
4. Назовите его «OpenClaw» и немедленно скопируйте токен — он больше не будет показан

## Установка Home Assistant MCP сервера

```bash
npm install -g @modelcontextprotocol/server-home-assistant
```

Проверьте работоспособность:

```bash
HA_URL=http://homeassistant.local:8123 \
HA_TOKEN=your_token_here \
npx @modelcontextprotocol/server-home-assistant
# Should start without errors
```

## Настройка OpenClaw

Добавьте MCP сервер в конфигурацию OpenClaw:

```yaml
# ~/.openclaw/config.yaml

mcpServers:
  home-assistant:
    command: npx
    args:
      - "@modelcontextprotocol/server-home-assistant"
    env:
      HA_URL: "http://homeassistant.local:8123"
      HA_TOKEN: "${HA_TOKEN}"
```

Задайте переменную окружения:

```bash
export HA_TOKEN="your_long_lived_access_token"
# Add to ~/.zshrc or ~/.bashrc to persist
```

Перезапустите OpenClaw:

```bash
openclaw restart
openclaw mcp list
# Should show: home-assistant (X tools)
```

## Доступные инструменты

HA MCP сервер предоставляет OpenClaw следующие инструменты:

```
get_states              - List all entity states
get_state               - Get a specific entity's state and attributes
call_service            - Call any HA service (turn on lights, etc.)
get_config              - Get HA configuration details
get_history             - Get historical state data for an entity
get_logbook             - Get logbook entries
list_automations        - List all automations
get_automation          - Get automation YAML
create_automation       - Create a new automation
get_template            - Evaluate a Jinja2 template
```

Это очень широкий набор возможностей — модель может делать практически всё то же, что вы делаете в интерфейсе HA.

## Базовые команды

После подключения команды на естественном языке работают сразу:

```
«Какая температура в гостиной?»
«Выключи весь свет»
«Установи термостат в спальне на 20°C»
«Кто-нибудь есть дома?»
«Закрой входную дверь на замок»
«Покажи уровень заряда всех датчиков»
```

Модель преобразует эти запросы в соответствующие вызовы сервисов. Для многосущностных команд вроде «выключи весь свет» она сначала вызывает `get_states`, чтобы найти все световые сущности, а затем вызывает `call_service` для нужных из них.

## Практические примеры

**Утренняя рутина по требованию:**

```
Вы: «Доброе утро, подготовь дом к утру — у меня есть примерно 45 минут до выхода»

OpenClaw: Настраиваю утреннюю рутину.
- Освещение на кухне → яркость 80%, тёплый белый
- Кофемашина → включить (если сущность доступна)
- Жалюзи в гостиной → открыть
- Термостат → 21°C
- Замок входной двери → проверен, закрыт

Что-нибудь ещё нужно для вашего утра?
```

**Диагностика проблем с устройствами:**

```
Вы: «Датчик движения в кабинете не срабатывал два дня, что происходит?»

OpenClaw: [Вызывает get_history для сущности датчика]

Просматриваю историю binary_sensor.office_motion:
- Последнее срабатывание: 2 дня назад в 9:47
- Заряд батареи: 8% (последний отчёт 3 дня назад)
- Текущее состояние: недоступен

Датчик перешёл в состояние «недоступен», а при заряде батареи 8% это скорее всего означает разряд. Замените батарейку — датчик должен переподключиться автоматически. Если этого не произойдёт в течение 10 минут, возможно, потребуется повторное сопряжение с координатором Zigbee/Z-Wave.
```

**Создание автоматизаций:**

```
Вы: «Создай автоматизацию, которая включает свет на крыльце при открытии входной двери после заката и выключает его через 5 минут после закрытия двери»

OpenClaw: [Вызывает list_automations для проверки конфликтов, затем create_automation]

Создана автоматизация «Свет на крыльце при открытии двери после заката»:
- Триггер: состояние front_door изменяется на 'open'
- Условие: солнце ниже горизонта
- Действие: включить porch_light
- Дополнительный триггер: состояние front_door изменяется на 'closed'
- Дополнительное действие: выключить porch_light через задержку 5 минут

Автоматизация активна. Проверить её работу, симулировав открытие двери?
```

**Запросы по мониторингу энергопотребления:**

```
Вы: «Какие устройства потребляют больше всего электроэнергии прямо сейчас?»

OpenClaw: [Вызывает get_states для всех сущностей мониторинга мощности]

Текущее потребление электроэнергии (от большего к меньшему):
1. Система HVAC: 2 340 Вт
2. Духовка (забыли выключить!): 1 800 Вт
3. Зарядная станция для электромобиля: 1 200 Вт
4. Холодильник: 145 Вт
5. Телевизор + развлекательная система: 87 Вт

Примечание: судя по всему, духовка включена — это намеренно? Суммарное текущее потребление: ~5,6 кВт.
```

## Вопросы безопасности

HA MCP сервер работает локально и напрямую взаимодействует с вашим экземпляром HA. Ключевые моменты безопасности:

**Ограничьте возможности OpenClaw.** Токен доступа HA по умолчанию даёт полный доступ к вашему экземпляру. Для более строгой настройки рассмотрите использование ограничения области действия API-ключей HA (доступно в некоторых версиях HA) или разверните прокси, который ограничивает вызываемые сервисы.

**Не открывайте внешний URL Home Assistant.** Настройте MCP сервер так, чтобы он использовал локальный адрес HA, а не внешний URL HA Cloud. Всё взаимодействие должно оставаться в локальной сети.

**Проверяйте автоматизации перед активацией.** Когда просите OpenClaw создать автоматизацию, просматривайте YAML до того, как она вступит в силу. Модель работает хорошо, но не безупречно — проверяйте граничные случаи.

```yaml
# More restrictive config: use local network only
mcpServers:
  home-assistant:
    env:
      HA_URL: "http://192.168.1.100:8123"  # Direct IP, not external URL
      HA_TOKEN: "${HA_TOKEN}"
```

**Ведите журнал действий OpenClaw.** В HA включите logbook для сущностей, которые вас интересуют. Если что-то произойдёт неожиданное, вы сможете отследить, какой сервис был вызван и когда.

## Продвинутый уровень: голосовые команды

Объедините интеграцию OpenClaw с MCP и поддержку голосового ввода для управления домом без рук:

```bash
# OpenClaw with voice input (when configured)
openclaw voice --provider home-assistant

# Or use the keyboard shortcut in OpenClaw's terminal UI
# Voice input → transcription → model processes → HA executes
```

Задержка от голосовой команды до действия составляет обычно 3–5 секунд: около 1 секунды на распознавание речи, 1–2 секунды на инференс модели, около 1 секунды на выполнение сервиса HA.

## Управление несколькими домами или зонами

Если у вас несколько экземпляров HA (основной дом + загородный дом), настройте несколько экземпляров MCP сервера:

```yaml
mcpServers:
  home-main:
    command: npx
    args: ["@modelcontextprotocol/server-home-assistant"]
    env:
      HA_URL: "http://homeassistant.local:8123"
      HA_TOKEN: "${HA_TOKEN_MAIN}"

  home-vacation:
    command: npx
    args: ["@modelcontextprotocol/server-home-assistant"]
    env:
      HA_URL: "https://vacation-home.duckdns.org:8123"
      HA_TOKEN: "${HA_TOKEN_VACATION}"
```

Модель может работать с обоими: «Проверь, не забыл ли кто-нибудь выключить свет на даче.»

## Часто задаваемые вопросы

**Будет ли это работать с моими устройствами Zigbee/Z-Wave/Matter/WiFi?**

Да. OpenClaw взаимодействует со слоем сущностей Home Assistant, который абстрагирует все протоколы устройств. Если устройство отображается как сущность в HA, OpenClaw может им управлять.

**Может ли OpenClaw случайно что-нибудь сломать?**

Модель вызывает реальные сервисы HA. Теоретически она может вызвать сервис, который вы не планировали. На практике модель осторожно относится к деструктивным действиям и обычно запрашивает подтверждение перед необратимыми операциями (например, отключением сигнализации). Всегда контролируйте неожиданные команды.

**Работает ли это с Home Assistant Cloud (Nabu Casa)?**

Да. Используйте внешний URL HA Cloud вместо локального адреса. Производительность будет немного ниже из-за дополнительного обращения через облако.

**Что если в моём экземпляре HA сотни сущностей?**

Вызов `get_states` возвращает все сущности, что может быть большим объёмом данных. Модель справляется с этим нормально, но ответы могут быть чуть медленнее. При очень большом количестве сущностей (500+) рекомендуется фильтровать по домену в командах: «покажи все световые сущности».

**Можно ли использовать это с офлайн-настройкой Ollama?**

Да. MCP сервер не зависит от бэкенда модели. Настройте OpenClaw на использование Ollama в качестве провайдера — HA MCP сервер останется доступным. В этом случае управление умным домом будет работать полностью в автономном режиме (при условии, что HA развёрнут локально).

Интеграция OpenClaw + Home Assistant изменила то, как я взаимодействую с домом. Создавать автоматизации через диалог намного естественнее, чем редактировать YAML вручную, а возможность задавать диагностические вопросы на обычном языке существенно экономит время на устранение неполадок. Если у вас уже есть установленный HA, настройка займёт около 15 минут.
