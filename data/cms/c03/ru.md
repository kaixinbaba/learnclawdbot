# Кейс C03: превращаем фрагменты экрана в Markdown для OpenClaw с помощью SNAG

## Профиль кейса

- **Категория:** Автоматизация / рабочий процесс разработчика
- **Для кого:** Пользователи, которые часто переносят фрагменты кода и UI из экрана в AI-чат
- **Источники:**
  - [OpenClaw Docs: Showcase](https://docs.openclaw.ai/start/showcase)
  - [am-will/snag](https://github.com/am-will/snag)

## Контекст

В задачах отладки и анализа пользователям регулярно нужно передавать в LLM небольшой участок интерфейса, кодовый фрагмент или диаграмму. Ручная перепечатка занимает время и увеличивает риск потерь.

Цель кейса — зафиксировать повторяемый путь:

- выделить область экрана,
- получить markdown,
- вставить результат в диалог OpenClaw.

## Что подтверждено источниками

По README SNAG и карточке в OpenClaw Showcase:

1. SNAG — это screenshot-to-text CLI.
2. Поддерживаются выделение области и работа с несколькими мониторами.
3. Инструмент обрабатывает текст/код/диаграммы/UI и копирует markdown-результат в буфер обмена.
4. Доступны провайдеры Google Gemini, OpenRouter и Z.AI.
5. SNAG представлен в OpenClaw Showcase как community-проект («Screenshot-to-Markdown»).

## Путь внедрения

### 1) Установить SNAG

```bash
uv tool install git+https://github.com/am-will/snag.git
```

### 2) Настроить провайдера и ключи API

```bash
snag --setup
```

Укажите provider/model/API key под вашу среду.

### 3) Выполнить захват и преобразование

```bash
snag
```

После выбора области SNAG кладёт markdown-вывод в буфер обмена.

### 4) Подключить к workflow OpenClaw

Вставьте результат из буфера в диалог OpenClaw и продолжайте отладку, резюмирование или review.

## Результат (на основе подтверждённых данных)

- Шаг «скриншот → текст» становится командным, а не ручным.
- Буфер обмена с готовым markdown уменьшает трение при сборке контекста для LLM.
- Поддержка нескольких провайдеров упрощает встраивание в существующий AI-стек.

## Подтверждённые факты и открытые вопросы

### ✅ Подтверждено источниками

- SNAG поддерживает выделение области и multi-monitor режим.
- SNAG поддерживает Gemini / OpenRouter / Z.AI.
- SNAG выдаёт markdown-готовый текст в буфер обмена.
- SNAG присутствует в OpenClaw Showcase.

### ⚠️ Требует дополнительной валидации (интервью/телеметрия)

- Средняя экономия времени на одну сессию захвата
- Изменение частоты ошибок относительно ручной перепечатки
- Влияние на долгосрочную скорость отладки

## Практические заметки

- На macOS при первом запуске потребуется разрешение на запись экрана.
- Для глобального хоткея команда `snag` должна быть доступна в PATH.
- Ключи API лучше хранить в локальном конфиге SNAG (`~/.config/snag/`).
