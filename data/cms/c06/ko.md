# Kev's Dream Team: OpenClaw로 14+ AI 에이전트 병렬 실행

## 개요

- **카테고리:** 멀티에이전트 오케스트레이션 / 생산성
- **대상:** AI 에이전트로 워크플로를 확장하려는 개인 개발자, 콘텐츠 크리에이터, 소규모 팀
- **출처:**
  - [OpenClaw 쇼케이스](https://docs.openclaw.ai/start/showcase)
  - [Orchestrated AI 아티클](https://github.com/adam91holt/orchestrated-ai-articles)

## 배경

단일 에이전트 워크플로는 복잡한 다단계 프로젝트를 처리할 때 한계에 달합니다. 콘텐츠 파이프라인 관리, 여러 출처에 걸친 연구, 또는 출판 워크플로 조정 여부와 관계없이 작업을 순차적으로 실행하면 병목 현상이 발생합니다.

이 사례에서는 OpenClaw의 서브에이전트 오케스트레이션을 사용하여 단일 개발자가 14개 이상의 AI 에이전트를 병렬로 실행하여 콘텐츠 중심 워크플로의 처리량을 극적으로 개선하는 방법을 살펴봅니다.

## OpenClaw 해결 방법

OpenClaw의 서브에이전트 시스템을 사용하면 단일 부모 에이전트에서 여러 동시 에이전트 세션을 생성할 수 있습니다. 주요 구성 옵션:

1. **`sessions_spawn`** - 구성에서 서브에이전트 생성 활성화
2. **`agents.defaults.subagents.maxConcurrent`** - 최대 동시 에이전트 수 설정 (예: 10-15)
3. **세션 오케스트레이션** - 부모 에이전트가 모든 서브에이전트의 결과 조정

### 단계별 설정

**1. 서브에이전트 생성 활성화**

OpenClaw 구성에 추가:

```yaml
agents:
  defaults:
    subagents:
      maxConcurrent: 14
sessions_spawn: true
```

**2. 에이전트 역할 정의**

워크플로의 다양한 역할에 대해 특수화된 프롬프트 생성:

- 연구 에이전트 - 여러 출처에서 정보 수집
- 작가 에이전트 - 초안 콘텐츠 작성
- 편집자 에이전트 - 콘텐츠 검토 및 개선
- SEO 에이전트 - 검색 엔진 최적화
- 출판 에이전트 - 배포 처리

**3. 워크플로 조정**

부모 에이전트가 모든 서브에이전트에 동시에 작업 전송:

```
부모: "주제 X 연구, 초안 Y 작성, Z 최적화를 병렬로 실행"
→ 서브에이전트 1: 주제 X 연구
→ 서브에이전트 2: 초안 Y 작성
→ 서브에이전트 3: Z 최적화
부모: 모든 결과 수집, 최종 출력 조정
```

## 결과

Orchestrated AI의 문서화된 워크플로 기반:

- **병렬 실행**으로 총 워크플로 시간 60~80% 단축
- **전문화된 에이전트**가 범용 에이전트보다 더 높은 품질의 출력 생성
- **확장 가능한 워크플로** - 아키텍처 변경 없이 다양한 작업에 더 많은 에이전트 추가 가능

## 주요 포인트

- 서브에이전트 오케스트레이션은 단일 에이전트 한계를 멀티에이전트.power로 변환
- 구성은 간단 - `sessions_spawn`을 활성화하고 동시성 제한 설정만 하면 됨
- 부모 에이전트는 지휘자처럼 전문화된 서브에이전트 조정
- 연구, 콘텐츠 제작, 다중 소스 분석 등 병렬화 가능한 작업에 가장 적합

## 확인된 사실 vs 검증 대기 중

### ✅ 출처에서 확인됨

- OpenClaw 쇼케이스는 멀티에이전트 오케스트레이션을 핵심 기능으로 소개
- `subagents` 구성이 병렬 에이전트 생성 활성화
- `maxConcurrent`가 동시에 실행할 수 있는 에이전트 수 제어

### ⚠️ 검증 대기 중 (개별 테스트 필요)

- 구체적인 처리량 향상은 워크플로 유형에 따라 다름
- 최적의 에이전트 수는 작업 복잡성과 API 속도 제한에 따라 다름

## 실용적인 참고 사항

- 5~10개의 동시 에이전트부터 시작하여 필요에 따라 조정
- 각 서브에이전트 역할에 대해 명확하고 고유한 프롬프트 사용
- 부모와 서브 에이전트 수준 모두에서 오류 처리 설정
- 병렬 에이전트가 더 많은 리소스를消耗하므로 API 사용량 모니터링
