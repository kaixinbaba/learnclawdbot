# OpenClaw + DeepSeek: идеальный бюджетный ИИ-ассистент

Если вы ищете способ запустить производительного ИИ-ассистента, не расходуя бюджет впустую, связка **OpenClaw** и **DeepSeek** — именно то, что нужно. Модели DeepSeek обеспечивают производительность, сопоставимую с лидерами рынка, при стоимости в разы ниже, чем у GPT-4o или Claude 3.5 Sonnet. А OpenClaw делает интеграцию максимально простой.

## Почему DeepSeek? Сравнение стоимости API

Цифры говорят сами за себя (данные начала 2026 года):

| Модель | Ввод (за 1M токенов) | Вывод (за 1M токенов) |
|---|---|---|
| DeepSeek-V3 | ~$0,27 | ~$1,10 |
| DeepSeek-R1 | ~$0,55 | ~$2,19 |
| GPT-4o | ~$2,50 | ~$10,00 |
| Claude 3.5 Sonnet | ~$3,00 | ~$15,00 |

Для большинства задач — помощи в написании кода, Q&A, суммаризации, работы с документами — DeepSeek-V3 выдаёт результаты, сопоставимые с крупными коммерческими моделями, примерно в **10 раз дешевле**. DeepSeek-R1 добавляет пошаговое рассуждение для задач, требующих глубокого анализа.

## Предварительные требования

Перед началом убедитесь, что у вас есть:

1. Активный **API-ключ DeepSeek** с [platform.deepseek.com](https://platform.deepseek.com)
2. Установленный и запущенный **OpenClaw** (рекомендуется v1.2+)
3. Node.js 18+ на сервере или локальной машине

## Руководство по настройке

Есть два способа подключить DeepSeek к OpenClaw: отредактировать файл `.env` напрямую или воспользоваться интерфейсом Dashboard.

### Способ 1: редактирование `.env`

Откройте корневую директорию OpenClaw и добавьте следующее в файл `.env`:

```bash
# Настройка провайдера DeepSeek
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
DEEPSEEK_MODEL=deepseek-chat        # Для DeepSeek-V3
# DEEPSEEK_MODEL=deepseek-reasoner  # Раскомментируйте для DeepSeek-R1
DEEPSEEK_BASE_URL=https://api.deepseek.com
```

Затем перезапустите OpenClaw:

```bash
npm run restart
# Если используете Docker:
docker compose restart openclaw
```

### Способ 2: через Dashboard

1. Перейдите на `http://localhost:3000` (или ваш адрес развёртывания)
2. Откройте **Настройки → ИИ-провайдеры**
3. Нажмите **Добавить провайдера** и выберите **DeepSeek**
4. Вставьте API-ключ и выберите модель
5. Нажмите **Сохранить и проверить** — при успешном подключении появится зелёная галочка

### Переключение между моделями

Вы можете задать отдельную модель для каждого скилла через поле `modelOverride`:

```json
{
  "skill": "code-review",
  "modelOverride": "deepseek-reasoner",
  "description": "Использовать R1 для глубокого анализа кода"
}
```

Мы протестировали DeepSeek-V3 на наборе типичных задач в OpenClaw:

**Задачи по коду** (генерация, отладка, объяснение): DeepSeek-V3 показал результаты наравне с GPT-4o для Python, JavaScript и SQL. R1 продемонстрировал заметное преимущество на многоэтапных алгоритмических задачах.

**Общие Q&A и суммаризация**: обе модели работают быстро (P50 задержка на типичных запросах менее 2 секунд) и точно справляются с фактическими запросами, суммаризацией документов и структурированным извлечением данных.

**Следование инструкциям**: V3 надёжно обрабатывает сложные многоэтапные инструкции. Для строгого форматирования вывода достаточно добавить явные указания по формату в системный промпт.

Практический вывод: **для подавляющего большинства сценариев использования OpenClaw DeepSeek-V3 — лучший по соотношению цена/качество провайдер на сегодняшний день.**

## Продвинутые советы: автоматическое переключение модели

OpenClaw поддерживает автоматическое переключение на резервный провайдер, чтобы ассистент оставался доступным при сбоях. Настройте в `config/providers.json`:

```json
{
  "failover": {
    "enabled": true,
    "order": ["deepseek", "openai", "anthropic"],
    "retryDelay": 1000,
    "maxRetries": 2
  }
}
```

Если DeepSeek временно недоступен, OpenClaw автоматически переключится на OpenAI или Anthropic, не возвращая ошибку пользователю.

Можно также настроить резервную модель для конкретного скилла:

```json
{
  "skill": "legal-draft",
  "primaryModel": "deepseek-chat",
  "failoverModel": "claude-3-5-sonnet",
  "failoverReason": "Резерв для контента с требованиями соответствия"
}
```
