# C05 пользовательский кейс: связать работу по задачам Linear с PR через linear-cli + OpenClaw

## Обзор

- **Категория:** Автоматизация / рабочий процесс разработки
- **Для кого:** Команды в Linear, которым нужен терминальный процесс с AI-координацией
- **Источники:**
  - [OpenClaw Showcase](https://docs.openclaw.ai/start/showcase)
  - [Finesssee/linear-cli](https://github.com/Finesssee/linear-cli)
  - [linear-cli examples](https://github.com/Finesssee/linear-cli/blob/master/docs/examples.md?raw=1)

## Контекст

Во многих командах работа по задачам распределена между браузером, локальным терминалом и чатами. Это увеличивает переключение контекста и снижает повторяемость рутины.

В этом кейсе используется документированный путь:

1. выполнять действия по задачам и git через `linear-cli`,
2. сохранять структурированный вывод,
3. передавать контекст в OpenClaw для следующих решений.

## Функции, подтверждённые источниками

По README проекта, examples-документации и записи в OpenClaw Showcase:

1. `linear-cli` покрывает широкий набор команд Linear (issues, projects, labels, teams, cycles, comments и др.).
2. Поддерживаются оба варианта аутентификации: OAuth через браузер и API key.
3. Для рабочего цикла задач доступны команды start/stop/close и операции с назначением.
4. Есть git-интеграция для веток и PR, связанных с задачей (`linear-cli g checkout`, `linear-cli g pr`).
5. Документированы форматы JSON/NDJSON для скриптов и agent-сценариев.
6. В OpenClaw Showcase Linear CLI указан как проект сообщества.

## Путь внедрения

### 1) Установка и проверка CLI

```bash
cargo install linear-cli
linear-cli --help
```

### 2) Настройка аутентификации

```bash
linear-cli auth oauth
# или
linear-cli config set-key lin_api_xxx
```

### 3) Выполнение потока задач из терминала

```bash
linear-cli i list --mine --output json --compact
linear-cli i start LIN-123 --checkout
linear-cli i comment LIN-123 -b "Work started from CLI workflow"
```

### 4) Переход к ветке/PR и продолжение в OpenClaw

```bash
linear-cli g pr LIN-123 --draft
```

Передайте вывод команд в диалог OpenClaw и продолжайте по чек-листу ревью, готовности к merge и следующим шагам.

## Результат (на основе подтверждённых данных)

- Поток от задачи до PR можно выполнять повторяемо через единый CLI-контур.
- Структурированный вывод упрощает автоматизацию и передачу контекста в AI.
- Для регулярных операций уменьшается зависимость от ручной навигации в UI.

## Подтверждённые факты и зоны допроверки

### ✅ Подтверждено источниками

- Linear CLI предоставляет команды для задач, git-процесса и операций рабочего пространства.
- Потоки OAuth/API key описаны в документации.
- Поддержка JSON/NDJSON описана в документации.
- Linear CLI присутствует в Showcase.

### ⚠️ Требует внутренних измерений

- Изменение среднего времени прохождения задачи
- Изменение задержки на этапе ревью после перехода на CLI-подход
- Долгосрочное влияние на пропускную способность в нескольких репозиториях

## Практические заметки

- Зафиксируйте единый формат идентификатора задач (например, `LIN-123`) в командном runbook.
- Для AI-цепочек используйте машинно-читаемый вывод.
- Явно задавайте способ аутентификации в каждом окружении, чтобы избежать путаницы рабочих пространств.
