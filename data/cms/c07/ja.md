# C07 ユーザーケース：SNAG - 画面キャプチャをLLM対応Markdownに変換

## ケース概要

- **カテゴリー：** DevTools / 開発ワークフロー
- **対象者：** 画面キャプチャを頻繁にテキストに変換してAI会話に使用する必要がある開発者
- **ソース：**
  - [OpenClaw ドキュメント：Showcase](https://docs.openclaw.ai/start/showcase)
  - [am-will/snag](https://github.com/am-will/snag)

## 背景

画面主体的ワークフローでは、開発者はUI要素、コードスニペット、ダイアグラムをLLM会話にコピーする必要があることがよくあります。手動での転記は遅く、エラーが発生しやすいです。

SNAGは合理化されたソリューションを提供します：

- クリックドラッグで任意の画面領域をキャプチャ
- AI駆動のMarkdown変換
- 自動クリップボード出力 - そのままペースト可能

## 検証済み機能

SNAG READMEとOpenClaw Showcaseより：

1. **領域選択**：クリックドラッグで画面の任意部分をキャプチャ
2. **マルチモニターサポート**：接続されたすべてのディスプレイに対応
3. **スマート文字起こし**：テキスト、コード、ダイアグラム、チャート、UI要素を処理
4. **インスタントクリップボード**：結果が自動コピー、そのままペースト可能
5. **複数のプロバイダー**：Google Gemini、OpenRouter、またはZ.AI (GLM-4.6V)
6. **クロスプラットフォーム**：Linux (X11/Wayland)、Windows、macOS

## 実装ステップ

### 1）SNAGをインストール

```bash
uv tool install git+https://github.com/am-will/snag.git
```

### 2）APIキーを設定

```bash
snag --setup
```

インタラクティブメニューが開きжды、希望のプロバイダーとAPIキーを設定できます。

### 3）画面をキャプチャ

```bash
snag
```

- 左クリック+ドラッグで領域を選択
- マウスを離してキャプチャと処理を実行
- 右クリックまたはEscapeでキャンセル

### 4）OpenClawにペースト

Markdown結果はすでにクリップボードにあります。デバッグ、要約、コードレビューのためにOpenClaw会話に直接ペーストしてください。

## 開発者生産性を向上させる理由

- **速度**：キャプチャからMarkdownまでは数秒（数分ではなく）
- **正確性**：AI文字起こしは複雑なレイアウト、コード構文、ダイアグラムを処理
- **クリップボードファースト**：ファイル保存不要 - インスタントペーストワークフロー
- **柔軟性**：ニーズに応じて異なるAIプロバイダーを選択可能

## 実践的なメモ

- **macOS**：初回実行時に画面権限を許可（システム設定 → 録画プライバシーとセキュリティ → 画面録画）
- **グローバルショートカット**：デスクトップ環境のショートカット（例：GNOMEのSuper+Shift+S）を設定してインスタントアクセス
- **APIキー**：`~/.config/shell環境変数にアクセスできないショートカットで動作
