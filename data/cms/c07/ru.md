# C07 Пользовательский кейс: SNAG - Преобразование скриншотов в Markdown для LLM

## Профиль кейса

- **Категория:** Инструменты разработчика / Рабочий процесс разработчика
- **Аудитория:** Разработчики, которым часто нужно преобразовывать скриншоты в текст для ИИ-диалогов
- **Источники:**
  - [Документация OpenClaw: Showcase](https://docs.openclaw.ai/start/showcase)
  - [am-will/snag](https://github.com/am-will/snag)

## Предыстория

При работе со скриншотами разработчикам часто нужно копировать элементы интерфейса, фрагменты кода или диаграммы в диалоги с LLM. Ручной перевод медленный и подвержен ошибкам.

SNAG предлагает оптимизированное решение:

- Захват любой области экрана кликом и перетаскиванием
- ИИ-преобразование в Markdown
- Автоматический вывод в буфер обмена - готовность к вставке

## Подтверждённые возможности

Из README SNAG и OpenClaw Showcase:

1. **Выбор области**: Клик и перетаскивание для захвата любой части экрана
2. **Поддержка нескольких мониторов**: Работает на всех подключённых дисплеях
3. **Умная транскрипция**: Обрабатывает текст, код, диаграммы, графики и элементы интерфейса
4. **Мгновенный буфер обмена**: Результат автоматически копируется, готов к вставке
5. **Несколько провайдеров**: Google Gemini, OpenRouter или Z.AI (GLM-4.6V)
6. **Кроссплатформенность**: Linux (X11/Wayland), Windows, macOS

## Шаги внедрения

### 1) Установить SNAG

```bash
uv tool install git+https://github.com/am-will/snag.git
```

### 2) Настроить API-ключи

```bash
snag --setup
```

Откроется интерактивное меню для настройки провайдера и API-ключа.

### 3) Захватить экран

```bash
snag
```

- Левый клик + перетаскивание для выбора области
- Отпустить мышь для захвата и обработки
- Правый клик или Escape для отмены

### 4) Вставить в OpenClaw

Результат в Markdown уже в буфере обмена. Вставьте его прямо в диалог OpenClaw для отладки, суммаризации или код-ревью.

## Почему это повышает продуктивность разработчика

- **Скорость**: Преобразование занимает секунды, а не минуты
- **Точность**: ИИ-транскрипция обрабатывает сложные макеты, синтаксис кода и диаграммы
- **Буфер обмена в приоритете**: Не требует сохранения файлов - мгновенный рабочий процесс вставки
- **Гибкость**: Выбор различных ИИ-провайдеров в зависимости от потребностей

## Практические notes

- **macOS**: При первом запуске предоставить разрешение на запись экрана (Настройки → Конфиденциальность и безопасность → Запись экрана)
- **Глобальные ярлыки**: Настройте ярлык в среде рабочего стола (например, Super+Shift+S в GNOME) для мгновенного доступа
- **API-ключи**: Хранятся в `~/.config/snag/.env` - работает с ярлыками без доступа к переменным окружения оболочки
