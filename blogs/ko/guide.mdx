# OpenClaw 초보자 가이드

OpenClaw에 오신 것을 환영합니다! 이 가이드는 실제로 일을 처리하는 AI 개인 비서를 설정하는 데 도움을 드립니다.

## OpenClaw란 무엇인가요?

OpenClaw는 좋아하는 메시징 앱을 통해 강력한 AI 에이전트에 연결하는 오픈소스 AI 개인 비서 게이트웨이입니다. 단순히 대화만 하는 전통적인 챗봇과 달리, OpenClaw는 **작업을 실행**하도록 설계되었습니다. 웹 브라우징, 스마트 홈 제어, 일정 관리, 코드 작성 등 훨씬 더 많은 작업을 수행할 수 있습니다.

### 핵심 철학: 대화보다 행동

대부분의 AI 어시스턴트는 답변을 제공하는 데 그칩니다. OpenClaw는 더 나아가 실제로 행동합니다. 이메일을 확인해야 하나요? OpenClaw가 할 수 있습니다. 브라우저를 자동화하고 싶으신가요? 완료. 코드를 배포해야 하나요? OpenClaw가 처리합니다. 이러한 근본적인 차이가 OpenClaw를 단순한 챗봇이 아닌 진정한 **개인 비서**로 만듭니다.

### 멀티 채널 지원

OpenClaw는 게이트웨이 역할을 하여 하나의 AI 에이전트를 여러 메시징 플랫폼에 연결합니다:

- **Telegram** — 설정이 쉽고 초보자에게 최적
- **Discord** — 커뮤니티와 팀에 완벽
- **WhatsApp** — 일상 메신저 사용
- **iMessage** — 네이티브 iOS/macOS 통합
- **Email** — 전통적이지만 강력
- 그 외 더 많은 플랫폼!

OpenClaw를 한 번 설치하면 AI 어시스턴트를 어디서든 채팅할 수 있는 곳에서 사용할 수 있습니다.

### 간단한 역사

OpenClaw는 여러 번의 반복을 거쳐 발전했습니다:
- **Moltbot** — 최초의 실험
- **Clawbot** — 첫 번째 안정 버전 릴리스
- **OpenClaw** — 현재의 오픈소스 버전

각 버전은 일상 워크플로우에 완벽하게 통합되는 개인 AI 에이전트를 제공한다는 핵심 아이디어를 개선했습니다.

## 주요 기능

### 멀티 채널 게이트웨이

하나의 OpenClaw 인스턴스로 여러 채팅 플랫폼을 동시에 서비스할 수 있습니다. 한 번 설정하면 Telegram, Discord 또는 WhatsApp에서 어시스턴트와 채팅할 수 있습니다. 대화와 컨텍스트가 어디든 따라옵니다.

### AI 에이전트 통합

OpenClaw는 여러 AI 제공업체를 지원합니다:
- **Anthropic Claude**(권장) — 뛰어난 추론 및 도구 사용 능력
- **OpenAI GPT** — 다재다능하고 강력함
- **Google Gemini** — 빠르고 멀티모달
- **Pi** — 대화형이고 친근함
- **커스텀 모델** — 자체 모델 사용

모델 간 전환하거나 다양한 목적으로 여러 에이전트를 실행할 수 있습니다.

### 브라우저 자동화

OpenClaw는 [Browser Relay](/blog/browser-relay) 기능을 통해 웹 브라우저를 제어할 수 있습니다. 양식 작성, 데이터 스크래핑 또는 반복적인 웹 작업 자동화가 필요하신가요? 에이전트가 여러분이 보는 동안 또는 백그라운드에서 수행할 수 있습니다.

### 스마트 홈 제어

**Home Assistant**와 통합하여 자연스러운 대화를 통해 조명, 온도 조절기, 카메라 등을 제어합니다. "거실 조명 끄기"가 간단하게 작동합니다.

### 개발자 워크플로우

OpenClaw는 개발자 작업에 탁월합니다:
- 코드 읽기 및 작성
- Git 작업(커밋, 푸시, PR 생성)
- 코드 리뷰
- CI/CD 파이프라인 관리
- SSH를 통한 서버 관리

### 스킬 시스템

OpenClaw의 기능은 **스킬 시스템**을 통해 확장 가능합니다. 각 스킬은 일정 관리부터 이미지 생성까지 특정 기능을 제공합니다. 고유한 요구에 맞는 커스텀 스킬을 만들 수 있습니다.

## 필요한 것

시작하기 전에 다음이 필요합니다:

1. **컴퓨터** - macOS, Linux 또는 Windows 실행
2. **Node.js 18+** 또는 **Docker** - OpenClaw 실행용
3. **AI 제공업체 API 키** — [Anthropic Claude](https://console.anthropic.com/)로 시작하는 것을 권장합니다
4. **메시징 앱 계정** — Telegram, Discord 또는 WhatsApp

그게 다입니다! 기본 설정은 놀랍도록 간단합니다.

## 빠른 설치

OpenClaw는 여러 설치 방법을 제공합니다. 다음은 간단한 개요입니다:

### Homebrew를 통한 설치(macOS/Linux)

```bash
brew install openclaw/tap/openclaw
```

### npm을 통한 설치

```bash
npm install -g openclaw
```

### Docker를 통한 설치

```bash
docker pull openclaw/openclaw:latest
```

### 소스에서 설치

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm run build
```

Windows 설정, 문제 해결 및 환경 구성을 포함한 자세한 설치 지침은 [완전한 설치 가이드](/blog/openclaw-setup-installation-guide)를 참조하세요.

## 첫 설정: 온보드 마법사

설치 후 OpenClaw는 시작을 돕는 대화형 설정 마법사를 제공합니다:

```bash
openclaw onboard
```

이 명령은 다음을 도와주는 친절한 안내를 시작합니다:

### 1. AI 제공업체 선택

마법사는 사용할 AI 서비스를 묻습니다. 초보자에게는 뛰어난 추론 및 도구 사용 능력을 가진 **Anthropic Claude**를 권장합니다.

### 2. API 키 설정

선택한 제공업체의 API 키를 제공해야 합니다:

- **Anthropic**: [console.anthropic.com](https://console.anthropic.com/)에서 받기
- **OpenAI**: [platform.openai.com](https://platform.openai.com/) 방문
- **Google**: [ai.google.dev](https://ai.google.dev/) 확인

마법사는 키를 구성 파일에 안전하게 저장합니다.

### 3. 첫 번째 채널 구성

다음으로 첫 번째 메시징 채널을 설정합니다. 마법사는 다음을 지원합니다:
- Telegram(초보자에게 가장 쉬움)
- Discord
- WhatsApp
- Email

하나를 선택하여 시작하세요. 나중에 더 많은 채널을 추가할 수 있습니다!

## 첫 번째 채널 연결

첫 번째 채널 설정을 단계별로 살펴보겠습니다. 가장 인기 있는 세 가지 옵션을 다룹니다.

### 옵션 A: Telegram(초보자에게 권장)

Telegram은 OpenClaw를 시작하는 가장 쉬운 방법입니다.

**1단계: BotFather로 봇 만들기**

1. Telegram을 열고 `@BotFather` 검색
2. `/newbot`을 전송하여 봇 만들기 시작
3. 이름 선택(예: "My OpenClaw Assistant")
4. 사용자 이름 선택('bot'으로 끝나야 함. 예: `my_openclaw_bot`)
5. BotFather가 **bot token**을 제공합니다 — 저장하세요!

**2단계: OpenClaw 구성**

```bash
openclaw channel add telegram
```

메시지가 표시되면 bot token을 붙여넣습니다.

**3단계: 채팅 시작**

1. Telegram에서 봇 찾기(만든 사용자 이름 검색)
2. `/start`를 전송하여 시작
3. "안녕하세요!"라고 말해보세요 — AI 어시스턴트가 응답할 것입니다!

### 옵션 B: Discord

Discord는 커뮤니티와 팀에 적합합니다.

**1단계: Discord 애플리케이션 만들기**

1. [discord.com/developers/applications](https://discord.com/developers/applications)로 이동
2. "New Application"을 클릭하고 이름 지정
3. "Bot" 탭으로 이동
4. "Add Bot"을 클릭하고 확인
5. "Token" 아래에서 "Reset Token"을 클릭하고 복사(안전하게 저장!)
6. 다음 특권 게이트웨이 인텐트 활성화:
   - Presence Intent
   - Server Members Intent
   - Message Content Intent

**2단계: OpenClaw 구성**

```bash
openclaw channel add discord
```

메시지가 표시되면 bot token을 붙여넣습니다.

**3단계: 서버에 초대**

1. Discord Developer Portal에서 "OAuth2" → "URL Generator"로 이동
2. 범위 선택: `bot` 및 `applications.commands`
3. 권한 선택: 최소한 `Read Messages/View Channels`, `Send Messages`, `Read Message History`
4. 생성된 URL을 복사하여 브라우저에서 열기
5. 서버를 선택하고 승인

**4단계: 채팅 시작**

봇이 액세스할 수 있는 모든 채널에서 멘션하거나 직접 DM하여 대화를 시작하세요!

### 옵션 C: WhatsApp

WhatsApp은 QR 코드 페어링이 필요하며 약간 복잡합니다.

**1단계: OpenClaw 구성**

```bash
openclaw channel add whatsapp
```

**2단계: QR 코드 스캔**

OpenClaw는 터미널에 QR 코드를 표시합니다. 휴대폰에서 WhatsApp을 열고 스캔합니다:

1. WhatsApp 열기
2. 메뉴 (⋮) 또는 설정 탭
3. "연결된 기기" 탭
4. "기기 연결" 탭
5. 터미널의 QR 코드 스캔

**3단계: 채팅 시작**

페어링 후 연결된 기기(자신)에게 메시지를 보내 연결을 테스트하세요!

WhatsApp 설정 및 문제 해결에 대한 자세한 내용은 [공식 문서](https://openclaw.org/docs)를 확인하세요.

## 첫 대화

이제 채널이 연결되었으니 AI 어시스턴트와 첫 대화를 나눠봅시다!

### 첫 메시지 보내기

간단한 것을 시도해보세요:

```
안녕하세요! 도와주실 수 있나요?
```

어시스턴트가 자신을 소개하고 무엇이 필요한지 물을 것입니다. OpenClaw 에이전트는 도움이 되고 친근하며 행동 지향적으로 설계되었습니다.

### 에이전트 응답 이해하기

어시스턴트에게 무언가를 요청하면 단계별로 작업을 생각하는 것을 알 수 있습니다. 예를 들어:

```
당신: AI에 대한 최신 뉴스를 웹에서 검색해줘
에이전트: 최근 AI 뉴스를 검색하겠습니다.
[에이전트가 web_search 도구 사용]
에이전트: 찾은 내용은 다음과 같습니다: [링크 및 요약이 포함된 결과]
```

에이전트는 무엇을 하고 있는지 알려주고, 적절한 도구를 사용하며, 유용한 결과를 반환합니다.

### 필수 명령

OpenClaw는 여러 내장 명령을 지원합니다:

**상태 확인:**
```
/status
```
현재 세션, 활성 모델 및 시스템 상태를 표시합니다.

**도움말 받기:**
```
/help
```
사용 가능한 명령 및 기본 사용 지침을 나열합니다.

**컨텍스트 보기:**
```
/context
```
현재 세션 컨텍스트와 메모리를 표시합니다.

**세션 재시작:**
```
/restart
```
현재 대화를 지우고 새로 시작합니다.

이러한 명령은 모든 채널에서 작동하며 어시스턴트를 관리하기 위한 필수 도구입니다.

## 기본 사항 이해하기

### 세션

OpenClaw와의 모든 대화는 **세션**에서 이루어집니다. 세션을 대화 컨텍스트로 생각하세요. 당신이 말한 것, 진행 중인 작업 및 관련 배경 정보를 기억합니다.

**주요 세션 개념:**

- **격리**: 각 채팅(DM, 채널, 그룹)에는 자체 세션이 있습니다
- **메모리**: 세션은 대화 기록을 기억합니다
- **컨텍스트**: 에이전트는 무엇에 대해 이야기하고 있었는지 알고 있습니다
- **지속성**: 세션은 재시작 후에도 유지됩니다(디스크에 저장됨)

에이전트가 무언가를 잊어버리거나 컨텍스트가 혼동되는 문제가 발생하면 문제 해결 팁은 [세션 메모리 가이드](/blog/troubleshooting/fix-session-memory-issues)를 확인하세요.

### 명령

위에서 기본 명령을 다뤘지만 OpenClaw는 더 많은 것을 지원합니다:

- `/thinking` — 추론 표시 전환
- `/model` — AI 모델 전환
- `/skills` — 사용 가능한 스킬 나열
- `/config` — 현재 구성 표시

전체 명령 참조는 [공식 문서](https://openclaw.org/docs/commands)를 참조하세요.

### 제어 UI

OpenClaw에는 설정을 관리하기 위한 웹 기반 제어판이 포함되어 있습니다.

**UI 액세스:**

```bash
openclaw start
```

그런 다음 브라우저에서 `http://localhost:3000`(기본 포트)을 엽니다.

**할 수 있는 것:**

- 모든 활성 세션 보기
- 대화 기록 보기
- 채널 구성 관리
- 시스템 상태 모니터링
- 에이전트 설정 조정
- 로그 및 디버깅 정보 보기

제어 UI는 여러 채널을 실행하거나 문제를 디버깅할 때 특히 유용합니다.

## 일반적인 작업

OpenClaw로 실제로 **무엇을 할 수 있는지** 살펴봅시다.

### 코드 지원

OpenClaw는 프로그래밍 작업 지원에 탁월합니다:

```
당신: CSV 파일을 읽고 두 번째 열의 숫자 평균을 계산하는 Python 스크립트를 만들어줘

에이전트: [코드를 생성하고 설명]
```

에이전트는 워크스페이스의 파일을 읽고, 새 코드를 작성하고, 복잡한 개념을 설명하고, 오류를 디버깅할 수도 있습니다.

### 일정 및 이메일 관리

적절한 스킬을 활성화하면:

```
당신: 내일 일정에 뭐가 있어?
에이전트: [일정을 확인하고 이벤트를 나열]

당신: john@example.com에게 회의에 대한 이메일을 보내줘
에이전트: [이메일 초안을 작성하고 전송 전 확인 요청]
```

### 웹 검색 및 연구

에이전트는 웹을 검색하고 발견 사항을 요약할 수 있습니다:

```
당신: React hooks의 모범 사례를 연구하고 요약해줘
에이전트: [검색, 기사 읽기, 포괄적인 요약 제공]
```

### 브라우저 자동화

[Browser Relay](/blog/browser-relay) 기능을 사용하면 에이전트가 웹 브라우저를 제어할 수 있습니다:

```
당신: example.com으로 가서 내 정보로 연락처 양식을 작성해줘
에이전트: [브라우저 열기, 탐색, 양식 작성 및 제출]
```

이는 반복적인 웹 작업을 자동화하거나 웹 애플리케이션을 테스트하는 데 매우 강력합니다.

### 스마트 홈 제어

Home Assistant를 통합한 경우:

```
당신: 모든 조명 꺼줘
에이전트: [Home Assistant에 명령 전송]

당신: 거실 온도가 어떻게 돼?
에이전트: [센서 확인 및 보고]
```

### 파일 관리 및 정리

```
당신: 다운로드 폴더의 모든 PDF 파일을 날짜별로 정리해줘
에이전트: [폴더 스캔, 날짜 기반 하위 폴더 생성, 파일 이동]
```

가능성은 무한합니다! 어시스턴트는 강력한 도구에 액세스할 수 있으며 이를 창의적으로 결합하여 복잡한 작업을 해결할 수 있습니다.

## 문제 해결

문제가 발생하셨나요? 일반적인 문제와 해결 방법에 대한 빠른 링크입니다:

### 인증 문제

API 인증 오류가 발생하는 경우:
- [Claude 인증 오류 수정](/blog/troubleshooting/fix-claude-authentication-errors)

### 도구 오류

"tool_use_id" 또는 유사한 오류가 표시되는 경우:
- [도구 사용 ID 오류 수정](/blog/troubleshooting/fix-tool-use-id-error)

### Cron 및 Heartbeat 문제

예약된 작업 또는 하트비트가 작동하지 않는 경우:
- [Cron/Heartbeat 문제 수정](/blog/troubleshooting/fix-cron-heartbeat-not-working)

### 세션 메모리 문제

에이전트가 무언가를 잊거나 컨텍스트가 혼동되는 경우:
- [세션 메모리 문제 수정](/blog/troubleshooting/fix-session-memory-issues)

### 일반 디버깅

일반적인 문제 해결:
1. 로그 확인: `openclaw logs`
2. 게이트웨이 재시작: `openclaw restart`
3. 구성 확인: `openclaw config show`
4. 시스템 상태 확인: 채팅에서 `/status` 명령

문제가 계속되면 [OpenClaw GitHub Issues](https://github.com/openclaw/openclaw/issues)를 방문하거나 커뮤니티 Discord에 참여하세요.
