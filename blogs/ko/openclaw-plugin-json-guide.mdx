# openclaw.plugin.json 완벽 가이드: 플러그인 매니페스트 설정

모든 OpenClaw 플러그인은 루트 디렉토리에 `openclaw.plugin.json` 매니페스트 파일을 **반드시** 포함해야 합니다. 이 파일은 OpenClaw가 코드를 실행하지 않고도 플러그인 설정을 검증하는 방법을 알려줍니다—보안과 신뢰성에 필수적입니다.

플러그인을 개발하거나 검증 오류를 해결하는 경우, 이 가이드에 필요한 모든 정보가 있습니다.

## 빠른 시작: 최소 매니페스트

가장 간단한 유효한 매니페스트입니다:

```json
\{
  "id": "my-plugin",
  "configSchema": \{
    "type": "object",
    "additionalProperties": false,
    "properties": \{\}
  \}
\}
```

이게 전부입니다—필수 필드는 두 개뿐. 하지만 할 수 있는 것은 훨씬 더 많습니다.

## 필수 필드

### `id` (문자열)

플러그인의 정식 식별자. 설치된 모든 플러그인 중에서 고유해야 합니다.

```json
\{
  "id": "voice-call"
\}
```

**규칙:**
- 소문자와 하이픈 사용 (예: `my-plugin`, `MyPlugin` 아님)
- 코어 도구 이름과 충돌하면 안 됨
- `plugins.entries.voice-call`과 같은 설정 참조에 사용됨

### `configSchema` (객체)

플러그인이 받아들이는 설정을 정의하는 JSON Schema. OpenClaw는 시작 시 이 스키마에 대해 플러그인 설정을 검증합니다—플러그인 코드를 실행하기 전에.

```json
\{
  "configSchema": \{
    "type": "object",
    "additionalProperties": false,
    "properties": \{
      "apiKey": \{
        "type": "string",
        "description": "서비스의 API 키"
      \},
      "timeout": \{
        "type": "number",
        "default": 30
      \}
    \},
    "required": ["apiKey"]
  \}
\}
```

**이것이 중요한 이유:**
- 시작 시 설정 오류를 즉시 포착
- 누락/잘못된 필드에 대한 명확한 오류 메시지 제공
- 향후 도구에서 설정 UI 생성 가능

## 선택 필드

### `name` (문자열)

사람이 읽을 수 있는 플러그인 표시 이름.

```json
\{
  "name": "Voice Call"
\}
```

### `description` (문자열)

플러그인이 하는 일에 대한 짧은 요약.

```json
\{
  "description": "Twilio/Telnyx/Plivo를 통한 발신 및 수신 음성 통화"
\}
```

### `version` (문자열)

플러그인 버전 (정보 제공용).

```json
\{
  "version": "1.2.0"
\}
```

### `kind` (문자열)

플러그인 카테고리. 예: 메모리 플러그인의 경우 `"memory"`.

```json
\{
  "kind": "memory"
\}
```

### `channels` (배열)

이 플러그인이 등록하는 채널 ID. 플러그인이 메시징 채널을 추가하는 경우 필수입니다.

```json
\{
  "channels": ["matrix", "slack"]
\}
```

**중요:** 여기에 선언되지 않은 설정의 `channels.*` 키는 **오류**입니다.

### `providers` (배열)

이 플러그인이 등록하는 제공자 ID (모델 제공자용).

```json
\{
  "providers": ["my-llm-provider"]
\}
```

### `skills` (배열)

로드할 스킬 디렉토리 (플러그인 루트 기준 상대 경로).

```json
\{
  "skills": ["skills/voice", "skills/transcribe"]
\}
```

### `uiHints` (객체)

설정 UI 렌더링 힌트—라벨, 플레이스홀더, 민감한 필드 마커.

```json
\{
  "uiHints": \{
    "apiKey": \{
      "label": "API 키",
      "placeholder": "sk-...",
      "sensitive": true
    \}
  \}
\}
```

## 전체 예제: Voice Call 플러그인

Voice Call 플러그인의 실제 매니페스트입니다:

```json
\{
  "id": "voice-call",
  "name": "Voice Call",
  "description": "Twilio/Telnyx/Plivo를 통한 발신/수신 통화",
  "version": "1.0.0",
  "configSchema": \{
    "type": "object",
    "additionalProperties": false,
    "properties": \{
      "provider": \{
        "type": "string",
        "enum": ["twilio", "telnyx", "plivo", "mock"],
        "description": "사용할 음성 제공자"
      \},
      "fromNumber": \{
        "type": "string",
        "description": "발신자 ID 전화번호"
      \},
      "toNumber": \{
        "type": "string",
        "description": "기본 수신 번호"
      \},
      "twilio": \{
        "type": "object",
        "properties": \{
          "accountSid": \{ "type": "string" \},
          "authToken": \{ "type": "string" \}
        \}
      \}
    \},
    "required": ["provider"]
  \},
  "uiHints": \{
    "twilio.authToken": \{
      "sensitive": true
    \}
  \}
\}
```

## 일반적인 오류 및 해결 방법

### "Plugin manifest not found"

**원인:** 플러그인 루트에 `openclaw.plugin.json`이 없습니다.

**해결:** 최소한 `id`와 `configSchema`가 포함된 매니페스트 파일을 만드세요.

### "Invalid configSchema"

**원인:** 스키마가 유효한 JSON Schema가 아닙니다.

**해결:** [jsonschema.dev](https://jsonschema.dev)에서 스키마를 검증하거나 JSON Schema 린터를 사용하세요.

### "Unknown channel: xxx"

**원인:** 설정이 어떤 플러그인에서도 선언되지 않은 채널 ID를 참조합니다.

**해결:** 매니페스트의 `channels` 배열에 채널 ID를 추가하세요.

### "Plugin ID conflicts with core tool"

**원인:** 플러그인 ID가 내장 도구 이름과 충돌합니다.

**해결:** 플러그인 ID를 고유한 이름으로 변경하세요.

## 검증 동작

OpenClaw는 다음 시점에 매니페스트를 검증합니다:

1. **설정 읽기/쓰기** — 모든 설정 변경 시 스키마 검증 실행
2. **Gateway 시작** — 모든 플러그인 매니페스트 로드 및 검증
3. **Doctor 명령** — `openclaw doctor`가 플러그인 오류 보고

플러그인 매니페스트에 문제가 있으면 검증 전체가 실패합니다—올바르게 설정하세요!

## 모범 사례

1. **항상 `additionalProperties: false` 사용** — 설정 키 오타 포착
2. **민감한 필드 표시** — 비밀에는 `uiHints.*.sensitive: true` 사용
3. **기본값 제공** — 선택 필드에는 스키마에서 `default` 사용
4. **설명 작성** — 사용자가 각 설정 옵션을 이해할 수 있도록
5. **Doctor로 테스트** — 변경 후 `openclaw doctor` 실행

## 플러그인 설정 사용

매니페스트를 설정하면 사용자는 `openclaw.json`에서 플러그인을 설정합니다:

```json5
\{
  plugins: \{
    entries: \{
      "voice-call": \{
        enabled: true,
        config: \{
          provider: "twilio",
          fromNumber: "+15550001234",
          twilio: \{
            accountSid: "ACxxxxxxxx",
            authToken: "..."
          \}
        \}
      \}
    \}
  \}
\}
```

OpenClaw가 `configSchema`에 대해 이 설정을 자동으로 검증합니다.

## 요약

`openclaw.plugin.json` 매니페스트는 플러그인과 OpenClaw 사이의 계약입니다:

| 필드 | 필수 | 용도 |
|------|------|------|
| `id` | ✅ | 고유 플러그인 식별자 |
| `configSchema` | ✅ | 검증용 JSON Schema |
| `name` | ❌ | 표시 이름 |
| `description` | ❌ | 짧은 설명 |
| `channels` | ❌ | 등록된 채널 ID |
| `providers` | ❌ | 등록된 제공자 ID |
| `skills` | ❌ | 로드할 스킬 디렉토리 |
| `uiHints` | ❌ | UI 렌더링 힌트 |

매니페스트를 올바르게 설정하면 OpenClaw가 나머지를 처리합니다—설정 검증, 조기 오류 감지, 플러그인 보안 유지.
