# OpenClaw Discord 통합: 완벽한 설정 가이드

Discord는 역할, 권한, 스레드, 포럼 채널 등 풍부한 기능을 제공하는 가장 강력한 봇 통합 플랫폼 중 하나입니다. OpenClaw 인스턴스를 Discord와 통합하면 단순한 메시징을 넘어 정교한 자동화, 멀티 서버 관리, 고급 권한 제어가 가능합니다.

이 포괄적인 가이드는 봇 생성 및 기본 권한 구성부터 스레드 관리 및 포럼 채널 지원과 같은 고급 기능 활용까지 전체 **openclaw discord setup** 프로세스를 안내합니다. 스레드 생성에 영향을 미치는 중요한 포럼 채널 함정(GitHub Issue #6601 기반)을 포함한 실제 문제도 다룹니다.

## Discord + OpenClaw를 선택하는 이유

Discord의 광범위한 API와 풍부한 기능 세트는 OpenClaw 통합에 이상적인 플랫폼입니다:

- **세분화된 권한**: 세밀한 보안을 위한 역할 기반 액세스 제어
- **풍부한 조직 구조**: 서버, 카테고리, 채널, 스레드, 포럼
- **확장성**: 여러 서버, 수백 개의 채널, 수천 명의 사용자 처리
- **네이티브 기능**: 리액션, 임베드, 버튼, 슬래시 명령 등
- **개발자 친화적**: 우수한 문서와 강력한 API

커뮤니티 서버 관리, 팀 조정 또는 사용자 정의 자동화 워크플로 구축 등 **openclaw discord permissions** 및 채널 관리를 통해 전례 없는 제어가 가능합니다.

## 사전 요구 사항

**openclaw discord setup**을 시작하기 전에 다음을 확인하세요:

- **OpenClaw 설치 및 실행 중** - 버전 0.5.0 이상 권장
- **Discord 계정** - 봇을 생성할 수 있는 권한
- **서버 관리자 액세스** - 테스트용으로 최소 하나의 Discord 서버
- **Discord 구조의 기본 이해** - 서버, 채널, 역할
- **HTTPS 엔드포인트(선택 사항)** - 프로덕션에서 웹훅 모드를 사용하는 경우에만 필요

## 파트 1: Discord 봇 만들기

### 1단계: Discord 개발자 포털 설정

**애플리케이션 만들기:**

1. [Discord 개발자 포털](https://discord.com/developers/applications)로 이동
2. **"New Application"** 클릭
3. 이름 입력 (예: "OpenClaw Assistant")
4. 개발자 서비스 약관 동의
5. **"Create"** 클릭

**봇 사용자 만들기:**

1. 애플리케이션에서 **"Bot"** 탭으로 이동
2. **"Add Bot"** 클릭
3. **"Yes, do it!"** 클릭하여 확인

**봇 토큰 얻기:**

1. Bot 탭에서 **"TOKEN"** 섹션 찾기
2. **"Reset Token"** 클릭 (처음이면 "Copy")
3. 토큰 복사 - 이것이 `DISCORD_BOT_TOKEN`입니다

**⚠️ 중요한 보안 경고**: 봇 토큰은 마스터 비밀번호와 같습니다. 이 토큰을 가진 사람은 누구나 봇을 제어할 수 있습니다. 버전 관리에 커밋하거나 공개적으로 공유하거나 스크린샷에 포함하지 마세요.

### 2단계: 필수 Intents 활성화

Discord는 특정 봇 기능을 위해 "특권 intents"를 명시적으로 활성화해야 합니다.

**OpenClaw에 필요한 Intents:**

Bot 탭에서 **"Privileged Gateway Intents"**로 스크롤하여 활성화:

1. **✅ Server Members Intent**
   - 용도: 사용자 조회, 허용 목록 매칭, 멤버 정보
   
2. **✅ Message Content Intent**
   - 용도: 길드에서 메시지 텍스트 읽기
   - 없으면: 봇이 연결되지만 메시지를 읽지 못하고 응답하지 않음
   - **이것이 "봇이 응답하지 않음" 문제의 1번 원인입니다**

3. **❌ Presence Intent** (선택 사항)
   - OpenClaw 기본 기능에는 필요하지 않음

**일반적인 실수**: 봇을 활성화했지만 Message Content Intent 활성화를 잊음. 이로 인해 봇이 온라인으로 보이지만 메시지에 응답하지 않습니다.

### 3단계: 봇 권한 구성

**권장 최소 권한:**

기본 **openclaw discord setup**을 위해 Bot 탭에서 이러한 권한을 활성화합니다:

- ✅ **View Channels** - 채널 목록 보기
- ✅ **Send Messages** - 사용자에게 응답
- ✅ **Read Message History** - 대화 컨텍스트 액세스
- ✅ **Embed Links** - 리치 임베드 전송
- ✅ **Attach Files** - 파일 및 이미지 공유
- ✅ **Add Reactions** - 메시지에 리액션
- ✅ **Use External Emojis** - 다른 서버의 커스텀 이모지 사용

**보안 모범 사례**: 실제로 필요한 권한만 부여하세요. 나중에 언제든지 추가할 수 있습니다.

### 4단계: 초대 URL 생성

**OAuth2 URL 생성기 사용:**

1. 애플리케이션에서 **"OAuth2"** → **"URL Generator"**로 이동

2. **Scopes** 선택:
   - ✅ `bot` - 핵심 봇 기능
   - ✅ `applications.commands` - 슬래시 명령 지원

3. **Bot Permissions** 선택:
   - 3단계에서 결정한 권한 체크

4. 생성된 URL 복사

**봇 초대:**

1. 브라우저에서 생성된 URL 열기
2. 봇을 추가할 서버 선택
3. 권한 검토
4. **"승인"** 클릭

## 파트 2: OpenClaw 구성

### 기본 Discord 구성

**구성 파일 설정:**

OpenClaw 구성 파일(일반적으로 `~/.openclaw/config.yaml` 또는 `~/.openclaw/config.json`) 편집:

```json
{
  "channels": {
    "discord": {
      "enabled": true,
      "token": "YOUR_BOT_TOKEN_HERE"
    }
  }
}
```

**환경 변수 사용 (프로덕션 권장):**

```bash
export DISCORD_BOT_TOKEN="YOUR_BOT_TOKEN_HERE"
```

**OpenClaw 시작:**

```bash
openclaw gateway start
```

로그에서 Discord 초기화 확인:

```
[INFO] Discord channel enabled
[INFO] Discord: Connected as @YourBotName#1234
```

### 액세스 제어 이해

OpenClaw는 두 가지 주요 정책을 통해 Discord 통합을 위한 정교한 액세스 제어를 제공합니다: DM 정책 및 그룹(길드) 정책.

**DM 정책 옵션:**

```json
{
  "channels": {
    "discord": {
      "dm": {
        "policy": "pairing",  // 권장 기본값
        "enabled": true
      }
    }
  }
}
```

**사용 가능한 DM 정책:**

1. **`pairing`** (권장) - 처음 사용자는 페어링 프로세스를 완료해야 함
2. **`allowlist`** - 사전 승인된 사용자만 봇에 DM 가능
3. **`open`** - 누구나 봇에 DM 가능
4. **`disabled`** - DM 기능 완전 비활성화

**그룹/길드 정책 옵션:**

```json
{
  "channels": {
    "discord": {
      "groupPolicy": "allowlist"  // 기본값
    }
  }
}
```

**사용 가능한 그룹 정책:**

1. **`allowlist`** (기본값) - 각 서버/채널에 대한 명시적 구성 필요
2. **`open`** - 추가된 모든 서버에서 봇이 응답
3. **`disabled`** - 모든 길드 메시지 무시

### 허용 목록 구성

**사용자 허용 목록 (DM):**

```json
{
  "channels": {
    "discord": {
      "dm": {
        "policy": "allowlist",
        "allowFrom": [
          "123456789012345678",  // 사용자 ID (권장)
          "steipete"             // 사용자 이름 (시작 시 해결)
        ]
      }
    }
  }
}
```

**Discord 사용자 ID 찾기:**

1. 개발자 모드 활성화: 사용자 설정 → 고급 → 개발자 모드
2. 어디서나 사용자 이름 우클릭 → 사용자 ID 복사

**길드 (서버) 허용 목록:**

```json
{
  "channels": {
    "discord": {
      "groupPolicy": "allowlist",
      "guilds": {
        "123456789012345678": {  // 길드 ID (숫자)
          "slug": "my-awesome-server",
          "users": ["987654321098765432"],
          "requireMention": true  // @멘션 시에만 응답
        }
      }
    }
  }
}
```

## 파트 3: 역할 및 권한 구성

### 채널별 구성

동일한 서버 내의 다른 채널에 대해 다른 동작을 구성합니다.

**예: 다중 채널 설정**

```json
{
  "channels": {
    "discord": {
      "guilds": {
        "123456789012345678": {
          "slug": "my-server",
          "requireMention": true,
          "users": ["987654321098765432"],
          
          "channels": {
            "general": {
              "allow": true,
              "requireMention": false
            },
            
            "help": {
              "allow": true,
              "requireMention": true,
              "users": ["111111111111111111"],
              "systemPrompt": "답변을 짧고 초보자 친화적으로 유지하세요."
            },
            
            "admin": {
              "allow": true,
              "users": ["987654321098765432"],
              "skills": ["ssh", "exec", "filesystem"],
              "systemPrompt": "완전한 시스템 액세스 권한이 있습니다. 주의하세요."
            }
          }
        }
      }
    }
  }
}
```

**채널 구성 옵션:**

- **`allow`**: `true`/`false` - 이 채널에서 봇이 응답하는지 여부
- **`requireMention`**: @멘션 시에만 봇이 응답
- **`users`**: 채널별 사용자 허용 목록
- **`skills`**: 사용 가능한 OpenClaw 스킬 제한
- **`systemPrompt`**: 채널별 AI 지침
- **`enabled`**: 채널 완전 비활성화

### 도구 및 액션 권한

```json
{
  "channels": {
    "discord": {
      "actions": {
        "reactions": true,
        "stickers": true,
        "polls": true,
        "messages": true,
        "threads": true,
        "pins": true,
        "search": true,
        
        // 높은 신뢰 액션 (기본적으로 비활성화)
        "roles": false,
        "moderation": false,
        "presence": false
      }
    }
  }
}
```

## 파트 4: 스레드 및 포럼 채널 지원

### Discord 스레드 유형 이해

Discord에는 세 가지 유형의 스레드가 있습니다:

1. **공개 스레드**: 텍스트 채널에서 생성, 모두에게 표시
2. **비공개 스레드**: 텍스트 채널 내 초대 전용 스레드
3. **포럼 스레드**: 포럼 채널의 게시물 (특수 케이스!)

### 기본 스레드 구성

```json
{
  "channels": {
    "discord": {
      "actions": {
        "threads": true
      }
    }
  }
}
```

**CLI를 통한 스레드 생성:**

```bash
openclaw message thread-create \
  --channel discord \
  --target channel:123456789012345678 \
  --thread-name "토론 주제"
```

### 포럼 채널 지원 (중요한 함정!)

**포럼 채널 함정 (GitHub Issue #6601):**

표준 `thread-create` 액션을 사용하여 포럼 채널에서 스레드를 생성하려고 하면 다음 오류가 발생할 수 있습니다:

```
Error: 포럼 채널에서 스레드를 생성할 수 없습니다
```

**왜 이런 일이 발생하는가:**

포럼 채널은 표준 `createThread` API 엔드포인트를 지원하지 않습니다. 대신 다른 서명과 요구 사항을 가진 `startThreadInForumChannel`을 사용해야 합니다.

**포럼 게시물을 생성하는 올바른 방법:**

```bash
openclaw message thread-create \
  --channel discord \
  --target channel:123456789012345678 \
  --thread-name "게시물 제목" \
  --message "첫 번째 메시지 내용"  # 포럼 게시물에 필수!
```

**주요 차이점**: 포럼 게시물에는 초기 메시지가 **필요**합니다. 일반 스레드는 비어 있을 수 있습니다.

**포럼별 구성:**

```json
{
  "channels": {
    "discord": {
      "guilds": {
        "123456789012345678": {
          "channels": {
            "forum-channel-id": {
              "allow": true,
              "requireMention": false,
              "systemPrompt": "이것은 포럼 채널입니다. 게시물을 만들 때 항상 주제를 설명하는 초기 메시지를 포함하세요."
            }
          }
        }
      }
    }
  }
}
```

## 파트 5: 일반적인 문제 및 문제 해결

### 문제 1: 봇이 메시지에 응답하지 않음

**증상:**
- 봇이 Discord에서 온라인으로 표시됨
- `openclaw status`가 Discord가 연결됨을 표시
- 길드 채널의 메시지에 응답하지 않음

**진단 단계:**

1. **Message Content Intent 확인:**
```bash
# Discord 개발자 포털 → 앱 → Bot → Privileged Gateway Intents
# "Message Content Intent"가 활성화되어 있는지 확인
```

활성화되지 않은 경우 활성화하고 OpenClaw 재시작:
```bash
openclaw gateway restart
```

2. **채널 권한 확인:**
```bash
openclaw message permissions --channel discord --target channel:YOUR_CHANNEL_ID
```

3. **OpenClaw 구성 확인:**
```bash
openclaw config get channels.discord.groupPolicy
```

`allowlist`인 경우 길드/채널이 구성되어 있는지 확인:
```bash
openclaw config get channels.discord.guilds
```

**솔루션:**

**솔루션 A**: Message Content Intent 활성화 (가장 일반적인 수정)
**솔루션 B**: 허용 목록에 길드/채널 추가
**솔루션 C**: 멘션 요구 사항 비활성화 (필요한 경우)

### 문제 2: 포럼 채널에서 스레드 생성 실패

**GitHub Issue #6601 기반**

**증상:**
- 일반 텍스트 채널에서는 `thread-create`가 작동
- 포럼 채널에서는 "잘못된 채널 유형" 오류로 실패

**솔루션:**

포럼 채널에는 다른 접근 방식이 필요합니다. 자세한 내용은 위의 포럼 채널 섹션을 참조하세요.

**빠른 수정:**

```bash
# 포럼 게시물을 만들 때 메시지 포함
openclaw message thread-create \
  --channel discord \
  --target channel:FORUM_CHANNEL_ID \
  --thread-name "게시물 제목" \
  --message "초기 게시물 내용"  # 이것은 필수입니다!
```

## 요약

이 가이드는 포괄적인 **openclaw discord setup**을 다루었습니다:

✅ **봇 생성**: 개발자 포털 설정, Intents, 권한
✅ **기본 구성**: 토큰 관리, 액세스 제어 정책
✅ **역할 및 권한**: 길드 수준, 채널 수준, 사용자 수준 허용 목록
✅ **스레드 및 포럼**: 표준 스레드, 포럼 채널, 중요한 포럼 함정
✅ **멀티 서버 설정**: 다른 구성으로 여러 길드 관리
✅ **문제 해결**: 일반적인 문제에 대한 실제 솔루션
✅ **보안**: 프로덕션 배포를 위한 모범 사례

**주요 사항:**

1. **Message Content Intent가 중요** - "봇이 응답하지 않음"의 1번 원인
2. **포럼 채널에는 특별한 처리 필요** - 메시지 내용 포함
3. **허용 목록 정책으로 시작** - 필요에 따라 점진적으로 개방
4. **프로덕션 구성에서 ID 사용** - Slug는 문서용
5. **최소한의 권한 부여** - 필요한 경우에만 추가

OpenClaw로 Discord를 즐기세요! 🤖💬
