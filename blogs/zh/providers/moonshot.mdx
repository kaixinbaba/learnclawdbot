Moonshot AI 的 Kimi 模型以超长上下文能力著称，擅长处理大量文档和长对话。本教程教你如何在 OpenClaw 中配置和使用 Kimi。

## 为什么选择 Moonshot？

- **超长上下文** — 支持高达 200K token 的上下文窗口，轻松处理长文档
- **中文优化** — 专门针对中文场景优化，理解和生成效果出色
- **国内直连** — 服务器在国内，无需翻墙，访问速度快

## 配置方式

运行 `openclaw onboard`，选择 Moonshot AI 进行配置。

{/* 截图：OpenClaw onboard 选择 Moonshot AI */}

### 获取 API Key

**1. 访问 [Moonshot AI 开放平台](https://platform.moonshot.cn/)，注册并登录：**

{/* 截图：Moonshot 开放平台首页 */}

**2. 进入控制台，点击左侧菜单的"API Key 管理"：**

{/* 截图：Moonshot 控制台侧边栏 */}

**3. 点击"新建"按钮创建 API Key：**

{/* 截图：API Key 列表页面，新建按钮 */}

**4. 填写 Key 名称（用于识别用途），点击确定：**

{/* 截图：创建 API Key 对话框 */}

**5. 复制生成的 API Key（只显示一次，请妥善保存）：**

{/* 截图：显示 API Key 的弹窗 */}

**6. 在 `openclaw onboard` 中选择 Moonshot AI，粘贴 API Key：**

{/* 截图：OpenClaw 粘贴 Moonshot API Key */}

### 配置示例

在 OpenClaw 配置文件中，Moonshot AI 的配置格式如下：

```yaml
models:
  moonshot:
    apiKey: YOUR_MOONSHOT_API_KEY
    baseURL: https://api.moonshot.cn/v1  # 可选，默认值
    models:
      - moonshot-v1-8k
      - moonshot-v1-32k
      - moonshot-v1-128k
```

## 模型选择

| 模型 | 上下文长度 | 特点 | 适用场景 |
|------|-----------|------|----------|
| moonshot-v1-8k | 8K tokens | 快速响应、成本低 | 日常对话、简短任务 |
| moonshot-v1-32k | 32K tokens | 平衡性价比 | 中等长度文档、一般任务 |
| moonshot-v1-128k | 128K tokens | 超长上下文 | 长文档分析、复杂对话 |

## 常见问题

### Kimi 和 Moonshot 是什么关系？

Kimi 是 Moonshot AI 推出的对话产品，Moonshot API 是其开放平台。在 OpenClaw 中使用的是 Moonshot API。

### 如何选择合适的上下文长度？

- **短对话、简单任务** → 8K 模型，速度快成本低
- **中等文档、多轮对话** → 32K 模型，性价比最优
- **长文档分析、超长对话** → 128K 模型，按需使用

### 免费额度是多少？

新注册用户通常有一定的免费试用额度，具体额度和计费规则请查看 Moonshot 官网最新政策。
