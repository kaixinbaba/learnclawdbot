一行命令安装，一行命令配置，然后就能用了。

OpenClaw（原名 ClawdBot / MoltBot）是一个开源 AI 助手框架，可以连接 Telegram、WhatsApp、Discord 等多个聊天平台，帮你处理日常任务。这篇教程带你从零开始，10 分钟跑起来。

## 前提条件

- **Node.js 22+**（必需，[安装指南](https://nodejs.org/)）
- **macOS / Linux / Windows（WSL2）**

> Windows 用户必须使用 [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install)，不支持原生 Windows 环境。

## 第一步：安装 OpenClaw

打开[终端](/blog/how-to-open-terminal)（不知道终端是什么？[点这里](/blog/how-to-open-terminal)），运行：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

脚本会自动检测系统环境、安装依赖、配置权限，全程无需手动干预。

安装完成后验证一下：

```bash
openclaw --version
```

> 📖 其他安装方式（npm 全局安装、源码安装等）请参考[官方安装文档](https://docs.openclaw.ai/install#quick-install-recommended)。

## 第二步：运行配置向导

```bash
openclaw onboard --install-daemon
```

<img src="/images/blog/install/agree.webp" alt="安全风险同意" style="max-width: 75%" />

向导会引导你完成所有必要设置：

**1. 模型认证**

选择 AI 模型提供商并登录：

**国际模型：**
[OpenAI](/blog/providers/openai) ·
[Anthropic (Claude)](/blog/providers/anthropic) ·
[Google (Gemini)](/blog/providers/google) ·

**本地模型：** [Ollama](/blog/openclaw-ollama)（零费用、完全离线）

<img src="/images/blog/install/onboard-model-auth.png" alt="OpenClaw 模型认证选择"  style="max-width: 45%" />

**2. 频道连接（可选）**

向导会询问是否立即连接聊天平台。你可以现在连接，也可以之后再配置：

- **Telegram** — 输入 Bot Token（从 [@BotFather](https://t.me/botfather) 获取）
- **WhatsApp** — 扫描二维码
- **Discord** — 输入 Bot Token

> 📖 各平台详细配置教程：[Telegram](/blog/integrations/telegram) · [WhatsApp](/blog/integrations/whatsapp) · [Discord](/blog/integrations/discord)

## 第三步：开始对话

配置完成后，你有两种方式和 OpenClaw 交互：

### 方式一：终端 TUI

```bash
openclaw tui
```

直接在终端里和 AI 对话，适合开发者和命令行爱好者。

<img src="/images/blog/install/tui-chat.png" alt="OpenClaw TUI 终端交互" />

### 方式二：Web Chat

打开浏览器访问：

```
http://localhost:18789
```

图形化聊天界面，体验更友好。

<img src="/images/blog/install/webchat.webp" alt="OpenClaw WebChat 界面" />

🎉 **搞定！** 你现在已经有一个运行中的 AI 助手了。

## 常见问题

### `openclaw` 命令找不到？

npm 全局路径不在 `PATH` 中。运行以下命令修复：

```bash
export PATH="$PATH:$(npm config get prefix)/bin"
source ~/.zshrc  # 或 ~/.bashrc
```

或者直接重新运行安装脚本，它会自动处理。

### Gateway 无法访问？

检查端口是否被占用：

```bash
openclaw gateway status
```
