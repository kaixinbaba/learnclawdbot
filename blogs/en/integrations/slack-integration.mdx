# Slack Integration for OpenClaw: Complete Setup Guide

Bring your AI assistant into Slack and collaborate with it across channels, DMs, and threads. OpenClaw integrates deeply with Slack's workspace model â€” supporting Socket Mode for easy setup, HTTP webhooks for production, and fine-grained access controls for enterprise environments.

This guide covers everything: from creating your Slack app to configuring channel permissions, threading behavior, slash commands, and multi-workspace deployments.

## How It Works

OpenClaw connects to Slack via the Slack API using either:

- **Socket Mode** (recommended): Real-time WebSocket connection, no public URL needed
- **HTTP Mode**: Events API webhooks for production deployments

Both modes support DMs, channels, threads, reactions, file uploads, and slash commands.

## Prerequisites

Before you begin:

- OpenClaw Gateway installed and running
- Slack workspace with admin access (to create apps)
- Understanding of Slack's channel vs DM model

## Quick Start (Socket Mode)

### Step 1: Create a Slack App

1. Go to [api.slack.com/apps](https://api.slack.com/apps)
2. Click **Create New App** â†’ **From scratch**
3. Name it "OpenClaw" and select your workspace

### Step 2: Enable Socket Mode

1. Go to **Settings â†’ Socket Mode**
2. Toggle **Enable Socket Mode** on
3. Go to **Basic Information â†’ App-Level Tokens**
4. Click **Generate Token and Scopes**
5. Add scope `connections:write`
6. Copy the **App Token** (`xapp-...`)

### Step 3: Configure OAuth & Permissions

1. Go to **OAuth & Permissions**
2. Add these **Bot Token Scopes**:
   - `chat:write` â€” send messages
   - `channels:history`, `channels:read` â€” read channels
   - `groups:history`, `groups:read` â€” read private channels
   - `im:history`, `im:read`, `im:write` â€” direct messages
   - `mpim:history`, `mpim:read`, `mpim:write` â€” group DMs
   - `users:read` â€” user info
   - `reactions:read`, `reactions:write` â€” reactions
   - `app_mentions:read` â€” @mentions
   - `commands` â€” slash commands
   - `files:read`, `files:write` â€” file handling
3. Click **Install to Workspace**
4. Copy the **Bot User OAuth Token** (`xoxb-...`)

### Step 4: Subscribe to Events

1. Go to **Event Subscriptions**
2. Toggle **Enable Events** on
3. Under **Subscribe to bot events**, add:
   - `message.channels`, `message.groups`, `message.im`, `message.mpim`
   - `app_mention`
   - `reaction_added`, `reaction_removed`
   - `member_joined_channel`, `member_left_channel`

### Step 5: Enable App Home

1. Go to **App Home**
2. Enable **Messages Tab** (so users can DM the bot)

### Step 6: Configure OpenClaw

Set tokens via environment variables:

```bash
export SLACK_APP_TOKEN="xapp-..."
export SLACK_BOT_TOKEN="xoxb-..."
```

Or in `~/.openclaw/openclaw.json`:

```json5
{
  channels: {
    slack: {
      enabled: true,
      appToken: "xapp-...",
      botToken: "xoxb-...",
    },
  },
}
```

### Step 7: Invite the Bot

Invite OpenClaw to channels:

```
/invite @OpenClaw
```

## DM Access Control

### Pairing Mode (Default, Secure)

Unknown DM senders receive a pairing code:

```json5
{
  channels: {
    slack: {
      dm: {
        policy: "pairing",
      },
    },
  },
}
```

**Approve senders:**

```bash
openclaw pairing list slack
openclaw pairing approve slack <CODE>
```

### Allowlist Mode

Only specific users can DM:

```json5
{
  channels: {
    slack: {
      dm: {
        policy: "allowlist",
        allowFrom: ["U123ABC", "@jane", "jane@company.com"],
      },
    },
  },
}
```

Accepts user IDs, @handles, or emails.

### Open Mode

Allow anyone in the workspace to DM:

```json5
{
  channels: {
    slack: {
      dm: {
        policy: "open",
        allowFrom: ["*"],
      },
    },
  },
}
```

## Channel Configuration

### Group Policy

Control which channels the bot can access:

```json5
{
  channels: {
    slack: {
      groupPolicy: "allowlist", // or "open", "disabled"
      channels: {
        "#general": { allow: true, requireMention: true },
        "#support": { allow: true, requireMention: false },
        "C123ABC": { allow: true },
      },
    },
  },
}
```

### Per-Channel Options

```json5
{
  channels: {
    slack: {
      channels: {
        "#engineering": {
          allow: true,
          requireMention: true,       // Only respond when @mentioned
          users: ["U123", "U456"],    // User allowlist for this channel
          skills: ["search", "docs"], // Limit available skills
          systemPrompt: "Keep answers technical and concise.",
          allowBots: false,           // Ignore bot messages (default)
        },
      },
    },
  },
}
```

## Reply Threading

Control how the bot threads replies:

```json5
{
  channels: {
    slack: {
      replyToMode: "off", // "off" | "first" | "all"
    },
  },
}
```

| Mode | Behavior |
|------|----------|
| `off` | Reply in main channel (default) |
| `first` | First reply threads, subsequent in channel |
| `all` | All replies go to thread |

### Per-Chat-Type Threading

```json5
{
  channels: {
    slack: {
      replyToMode: "off",
      replyToModeByChatType: {
        direct: "all",    // Thread all DMs
        group: "first",   // Thread first MPIM reply
        channel: "off",   // Keep channels in main
      },
    },
  },
}
```

## Slash Commands

### Built-in Command

Configure a `/openclaw` slash command:

```json5
{
  channels: {
    slack: {
      slashCommand: {
        enabled: true,
        name: "openclaw",
        ephemeral: true, // Only visible to the user
      },
    },
  },
}
```

In your Slack app, create a slash command pointing to your webhook URL.

### Native Commands

Enable OpenClaw's built-in commands as Slack slash commands:

```json5
{
  channels: {
    slack: {
      commands: {
        native: true,
      },
    },
  },
}
```

## User Token (Optional)

For enhanced read access, add a user token:

```json5
{
  channels: {
    slack: {
      botToken: "xoxb-...",
      appToken: "xapp-...",
      userToken: "xoxp-...",
      userTokenReadOnly: true, // Default: only use for reads
    },
  },
}
```

User tokens enable:
- Search across workspace
- Access to more conversation history
- Enhanced member info

**Security note:** User tokens have the installing user's permissions. Keep `userTokenReadOnly: true` unless you need writes.

## Troubleshooting

### Bot not responding

1. Check tokens are correct (`SLACK_APP_TOKEN`, `SLACK_BOT_TOKEN`)
2. Verify Socket Mode is enabled in Slack app settings
3. Ensure bot is invited to the channel
4. Check `groupPolicy` allows the channel

### "Missing scope" errors

Add the required scope in **OAuth & Permissions** and reinstall the app.

### Messages not received

1. Check **Event Subscriptions** has the right events
2. For HTTP mode, verify webhook URL is reachable
3. Check `dm.policy` and `dm.allowFrom` for DMs

### Bot responds to itself

Set `allowBots: false` (default) to ignore bot messages:

```json5
{
  channels: {
    slack: {
      channels: {
        "#general": { allowBots: false },
      },
    },
  },
}
```

## Security Best Practices

- Use **pairing mode** for DMs in public workspaces
- Set **channel allowlists** to limit bot access
- Keep **userTokenReadOnly: true** unless you need writes
- Use **requireMention: true** in busy channels
- Review **actions** gates to limit bot capabilities
