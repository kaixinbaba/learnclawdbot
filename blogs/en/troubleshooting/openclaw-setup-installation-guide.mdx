# Complete Guide: Setting Up OpenClaw on Windows, macOS, and Linux

Getting OpenClaw running on your system should be straightforward, but different operating systems have unique quirks. This guide walks you through installation on **Windows**, **macOS**, and **Linux**, covering common pitfalls and fixes.

## Prerequisites: What You Need First

Before installing OpenClaw, ensure you have:

### 1. Node.js (v18 or higher)

OpenClaw requires a modern Node.js runtime. Check your version:

```bash
node --version
```

If you don't have Node.js or have an older version, install it:

**macOS (using Homebrew):**
```bash
brew install node
```

**Windows (using winget):**
```powershell
winget install OpenJS.NodeJS
```

**Linux (Ubuntu/Debian):**
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**Linux (Fedora/RHEL):**
```bash
sudo dnf install nodejs
```

### 2. npm or pnpm

npm comes with Node.js, but pnpm is faster and more efficient:

```bash
npm install -g pnpm
```

## Installing OpenClaw

### Step 1: Install via npm

```bash
npm install -g @anthropic-ai/openclaw
```

Or with pnpm:

```bash
pnpm add -g @anthropic-ai/openclaw
```

### Step 2: Verify Installation

```bash
openclaw --version
```

You should see the version number (e.g., `0.5.2`).

## Platform-Specific Setup

### ü™ü Windows

#### Issue #1: PowerShell Execution Policy

**Symptom:** `openclaw: File ... cannot be loaded because running scripts is disabled`

**Fix:**
```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

#### Issue #2: Long Path Support

Windows has a 260-character path limit that can break npm installs.

**Fix:** Enable long paths in registry
```powershell
New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" -Name "LongPathsEnabled" -Value 1 -PropertyType DWORD -Force
```

Or use WSL2 (Windows Subsystem for Linux):
```powershell
wsl --install
```

Then install OpenClaw inside WSL:
```bash
# Inside WSL
npm install -g @anthropic-ai/openclaw
```

#### Issue #3: Node-gyp Compilation Errors

**Symptom:** Errors during installation about missing build tools

**Fix:** Install Windows Build Tools
```powershell
npm install -g windows-build-tools
```

Or install Visual Studio Build Tools manually from Microsoft.

### üçé macOS

#### Issue #1: Permission Denied

**Symptom:** `EACCES: permission denied` when running `npm install -g`

**Fix:** Never use `sudo` with npm. Instead, configure npm to use a user directory:

```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc
```

Then reinstall:
```bash
npm install -g @anthropic-ai/openclaw
```

#### Issue #2: Command Not Found After Install

**Symptom:** `openclaw: command not found` even after successful install

**Fix:** Check if npm global bin is in PATH:
```bash
echo $PATH | grep npm
```

If missing, add to `~/.zshrc`:
```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

#### Issue #3: Rosetta Required (M1/M2 Macs)

Some dependencies may need Rosetta 2 on Apple Silicon:

```bash
softwareupdate --install-rosetta
```

### üêß Linux

#### Issue #1: EACCES During Global Install

**Symptom:** Permission errors like macOS

**Fix:** Same solution - use a user directory:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @anthropic-ai/openclaw
```

#### Issue #2: Missing Dependencies (Ubuntu/Debian)

**Symptom:** Build errors for native modules

**Fix:**
```bash
sudo apt-get update
sudo apt-get install -y build-essential python3
```

#### Issue #3: Snap Node.js Issues

If you installed Node.js via Snap, you may face permission issues.

**Fix:** Uninstall Snap version and use official repo:
```bash
sudo snap remove node
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

## First-Time Configuration

After installation, configure OpenClaw:

### 1. Set Up API Key

```bash
openclaw configure
```

Follow the prompts to add your Anthropic API key.

### 2. Start the Gateway

```bash
openclaw gateway start
```

### 3. Test the Setup

```bash
openclaw chat "Hello, OpenClaw!"
```

If you see a response, congratulations! üéâ

## Common Post-Install Issues

### Gateway Won't Start

**Check logs:**
```bash
tail -f ~/.openclaw/logs/gateway.log
```

**Common fixes:**
- Port 3000 already in use ‚Üí Change port in config
- Missing config file ‚Üí Run `openclaw configure` again
- Firewall blocking ‚Üí Allow Node.js in firewall settings

### Outdated Version

Update OpenClaw:
```bash
npm update -g @anthropic-ai/openclaw
```

Or with pnpm:
```bash
pnpm update -g @anthropic-ai/openclaw
```

### Complete Reinstall

If all else fails:
```bash
# Uninstall
npm uninstall -g @anthropic-ai/openclaw

# Clear cache
npm cache clean --force

# Reinstall
npm install -g @anthropic-ai/openclaw
```
