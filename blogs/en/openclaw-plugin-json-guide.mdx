# openclaw.plugin.json: Complete Guide to Plugin Manifest Configuration

Every OpenClaw plugin **must** include an `openclaw.plugin.json` manifest file in its root directory. This file tells OpenClaw how to validate your plugin's configuration without executing any code—making it essential for security and reliability.

If you're building a plugin or troubleshooting validation errors, this guide covers everything you need to know.

## Quick Start: Minimal Manifest

Here's the simplest valid manifest:

```json
{
  "id": "my-plugin",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

That's it—two required fields. But there's much more you can do.

## Required Fields

### `id` (string)

The canonical plugin identifier. Must be unique across all installed plugins.

```json
{
  "id": "voice-call"
}
```

**Rules:**
- Use lowercase with hyphens (e.g., `my-plugin`, not `MyPlugin`)
- Cannot clash with core tool names
- Used in config references like `plugins.entries.voice-call`

### `configSchema` (object)

A JSON Schema that defines what configuration your plugin accepts. OpenClaw validates plugin config against this schema at startup—before running any plugin code.

```json
{
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "apiKey": {
        "type": "string",
        "description": "API key for the service"
      },
      "timeout": {
        "type": "number",
        "default": 30
      }
    },
    "required": ["apiKey"]
  }
}
```

**Why this matters:**
- Catches config errors immediately at startup
- Provides clear error messages for missing/invalid fields
- Enables config UI generation in future tools

## Optional Fields

### `name` (string)

Human-readable display name for your plugin.

```json
{
  "name": "Voice Call"
}
```

### `description` (string)

Short summary of what the plugin does.

```json
{
  "description": "Outbound and inbound voice calls via Twilio/Telnyx/Plivo"
}
```

### `version` (string)

Plugin version (informational only).

```json
{
  "version": "1.2.0"
}
```

### `kind` (string)

Plugin category. Example: `"memory"` for memory plugins.

```json
{
  "kind": "memory"
}
```

### `channels` (array)

Channel IDs this plugin registers. Required if your plugin adds a messaging channel.

```json
{
  "channels": ["matrix", "slack"]
}
```

**Important:** Unknown `channels.*` keys in config are **errors** unless declared here.

### `providers` (array)

Provider IDs this plugin registers (for model providers).

```json
{
  "providers": ["my-llm-provider"]
}
```

### `skills` (array)

Skill directories to load, relative to the plugin root.

```json
{
  "skills": ["skills/voice", "skills/transcribe"]
}
```

### `uiHints` (object)

Hints for config UI rendering—labels, placeholders, and sensitive field markers.

```json
{
  "uiHints": {
    "apiKey": {
      "label": "API Key",
      "placeholder": "sk-...",
      "sensitive": true
    }
  }
}
```

## Complete Example: Voice Call Plugin

Here's a real-world manifest from the Voice Call plugin:

```json
{
  "id": "voice-call",
  "name": "Voice Call",
  "description": "Outbound + inbound calls via Twilio/Telnyx/Plivo",
  "version": "1.0.0",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "provider": {
        "type": "string",
        "enum": ["twilio", "telnyx", "plivo", "mock"],
        "description": "Voice provider to use"
      },
      "fromNumber": {
        "type": "string",
        "description": "Caller ID phone number"
      },
      "toNumber": {
        "type": "string",
        "description": "Default destination number"
      },
      "twilio": {
        "type": "object",
        "properties": {
          "accountSid": { "type": "string" },
          "authToken": { "type": "string" }
        }
      }
    },
    "required": ["provider"]
  },
  "uiHints": {
    "twilio.authToken": {
      "sensitive": true
    }
  }
}
```

## Common Errors and Fixes

### "Plugin manifest not found"

**Cause:** Missing `openclaw.plugin.json` in plugin root.

**Fix:** Create the manifest file with at least `id` and `configSchema`.

### "Invalid configSchema"

**Cause:** The schema isn't valid JSON Schema.

**Fix:** Validate your schema at [jsonschema.dev](https://jsonschema.dev) or use a JSON Schema linter.

### "Unknown channel: xxx"

**Cause:** Config references a channel ID not declared by any plugin.

**Fix:** Add the channel ID to your manifest's `channels` array.

### "Plugin ID conflicts with core tool"

**Cause:** Your plugin ID matches a built-in tool name.

**Fix:** Rename your plugin ID to something unique.

## Validation Behavior

OpenClaw validates manifests at these points:

1. **Config read/write** — Schema validation runs on every config change
2. **Gateway startup** — All plugin manifests are loaded and validated
3. **Doctor command** — `openclaw doctor` reports plugin errors

If a plugin has a broken manifest, validation fails entirely—so get it right!

## Best Practices

1. **Always use `additionalProperties: false`** — Catches typos in config keys
2. **Mark sensitive fields** — Use `uiHints.*.sensitive: true` for secrets
3. **Provide defaults** — Use `default` in schema for optional fields
4. **Write descriptions** — Help users understand each config option
5. **Test with Doctor** — Run `openclaw doctor` after changes

## Using Your Plugin Config

Once your manifest is set up, users configure your plugin in their `openclaw.json`:

```json5
{
  plugins: {
    entries: {
      "voice-call": {
        enabled: true,
        config: {
          provider: "twilio",
          fromNumber: "+15550001234",
          twilio: {
            accountSid: "ACxxxxxxxx",
            authToken: "..."
          }
        }
      }
    }
  }
}
```

OpenClaw validates this config against your `configSchema` automatically.

## Summary

The `openclaw.plugin.json` manifest is your plugin's contract with OpenClaw:

| Field | Required | Purpose |
|-------|----------|---------|
| `id` | ✅ | Unique plugin identifier |
| `configSchema` | ✅ | JSON Schema for validation |
| `name` | ❌ | Display name |
| `description` | ❌ | Short summary |
| `channels` | ❌ | Registered channel IDs |
| `providers` | ❌ | Registered provider IDs |
| `skills` | ❌ | Skill directories to load |
| `uiHints` | ❌ | UI rendering hints |

Get your manifest right, and OpenClaw handles the rest—validating config, catching errors early, and keeping your plugin secure.
