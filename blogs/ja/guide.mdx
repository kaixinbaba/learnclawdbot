# OpenClaw 初心者ガイド

OpenClaw へようこそ！このガイドでは、実際に仕事をこなす AI パーソナルアシスタントのセットアップを支援します。

## OpenClaw とは？

OpenClaw は、お気に入りのメッセージングアプリを通じて強力な AI エージェントに接続する、オープンソースの AI パーソナルアシスタントゲートウェイです。従来のチャットボットが単に話すだけなのとは異なり、OpenClaw は**タスクを実行する**ように設計されています。ウェブの閲覧、スマートホームの制御、カレンダーの管理、コードの作成など、さまざまなことができます。

### 核心理念：会話より行動

ほとんどの AI アシスタントは答えを提供するだけで終わります。OpenClaw はさらに進んで、実際に行動します。メールをチェックする必要がありますか？OpenClaw ができます。ブラウザを自動化したいですか？完了。コードをデプロイする必要がありますか？OpenClaw が処理します。この根本的な違いにより、OpenClaw は単なるチャットボットではなく、真の**パーソナルアシスタント**になります。

### マルチチャネルサポート

OpenClaw はゲートウェイとして機能し、1つの AI エージェントを複数のメッセージングプラットフォームに接続します：

- **Telegram** — セットアップが簡単、初心者に最適
- **Discord** — コミュニティやチームに最適
- **WhatsApp** — 日常のメッセンジャーを使用
- **iMessage** — ネイティブ iOS/macOS 統合
- **Email** — 伝統的だが強力
- その他多数！

OpenClaw を一度インストールすれば、AI アシスタントがあらゆる場所で利用できるようになります。

### 簡単な歴史

OpenClaw はいくつかの反復を経て進化してきました：
- **Moltbot** — 最初の実験
- **Clawbot** — 最初の安定版リリース
- **OpenClaw** — 現在のオープンソース版

各バージョンは、日常のワークフローにシームレスに統合されるパーソナル AI エージェントを提供するという核心的なアイデアを改善してきました。

## 主な機能

### マルチチャネルゲートウェイ

1つの OpenClaw インスタンスで複数のチャットプラットフォームに同時にサービスを提供できます。一度セットアップすれば、Telegram、Discord、WhatsApp からアシスタントとチャットできます。会話とコンテキストはどこにでも付いてきます。

### AI エージェント統合

OpenClaw は複数の AI プロバイダーをサポートしています：
- **Anthropic Claude**（推奨）— 優れた推論とツール使用能力
- **OpenAI GPT** — 多用途で強力
- **Google Gemini** — 高速でマルチモーダル
- **Pi** — 会話的で親しみやすい
- **カスタムモデル** — 独自のモデルを使用

モデル間で切り替えたり、異なる目的で複数のエージェントを実行したりできます。

### ブラウザ自動化

OpenClaw は [Browser Relay](/blog/browser-relay) 機能を通じてウェブブラウザを制御できます。フォームの記入、データのスクレイピング、繰り返しのウェブタスクの自動化が必要ですか？エージェントがあなたが見ている間またはバックグラウンドで実行できます。

### スマートホーム制御

**Home Assistant** と統合して、自然な会話で照明、サーモスタット、カメラなどを制御します。「リビングルームのライトを消して」が簡単に機能します。

### 開発者ワークフロー

OpenClaw は開発者タスクに優れています：
- コードの読み書き
- Git 操作（コミット、プッシュ、PR 作成）
- コードレビュー
- CI/CD パイプライン管理
- SSH 経由のサーバー管理

### スキルシステム

OpenClaw の機能は**スキルシステム**を通じて拡張可能です。各スキルは特定の機能を提供します。カレンダー管理から画像生成まで。独自のニーズに合わせてカスタムスキルを作成できます。

## 必要なもの

開始する前に、以下が必要です：

1. **コンピューター** - macOS、Linux、または Windows
2. **Node.js 18+** または **Docker** - OpenClaw を実行するため
3. **AI プロバイダー API キー** — [Anthropic Claude](https://console.anthropic.com/) から始めることをお勧めします
4. **メッセージングアプリアカウント** — Telegram、Discord、または WhatsApp

これだけです！基本的なセットアップは驚くほど簡単です。

## クイックインストール

OpenClaw はいくつかのインストール方法を提供しています。以下は簡単な概要です：

### Homebrew 経由（macOS/Linux）

```bash
brew install openclaw/tap/openclaw
```

### npm 経由

```bash
npm install -g openclaw
```

### Docker 経由

```bash
docker pull openclaw/openclaw:latest
```

### ソースから

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm run build
```

Windows セットアップ、トラブルシューティング、環境設定を含む詳細なインストール手順については、[完全インストールガイド](/blog/openclaw-setup-installation-guide)を参照してください。

## 初回セットアップ：オンボードウィザード

インストール後、OpenClaw は開始を支援するインタラクティブなセットアップウィザードを提供します：

```bash
openclaw onboard
```

このコマンドは、以下を支援するフレンドリーなウォークスルーを起動します：

### 1. AI プロバイダーを選択

ウィザードは、使用したい AI サービスを尋ねます。初心者には、優れた推論とツール使用能力を持つ **Anthropic Claude** をお勧めします。

### 2. API キーを設定

選択したプロバイダーから API キーを提供する必要があります：

- **Anthropic**：[console.anthropic.com](https://console.anthropic.com/) で取得
- **OpenAI**：[platform.openai.com](https://platform.openai.com/) を訪問
- **Google**：[ai.google.dev](https://ai.google.dev/) を確認

ウィザードはキーを設定ファイルに安全に保存します。

### 3. 最初のチャネルを設定

次に、最初のメッセージングチャネルを設定します。ウィザードは以下をサポートしています：
- Telegram（初心者に最適）
- Discord
- WhatsApp
- Email

1つを選択して開始してください。後で追加のチャネルを追加できます！

## 最初のチャネルを接続

最初のチャネルのセットアップを順を追って見ていきましょう。最も人気のある3つのオプションをカバーします。

### オプション A：Telegram（初心者に推奨）

Telegram は OpenClaw を始める最も簡単な方法です。

**ステップ 1：BotFather でボットを作成**

1. Telegram を開き、`@BotFather` を検索
2. `/newbot` を送信してボットの作成を開始
3. 名前を選択（例：「My OpenClaw Assistant」）
4. ユーザー名を選択（'bot' で終わる必要があります。例：`my_openclaw_bot`）
5. BotFather が **bot token** を提供します — 保存してください！

**ステップ 2：OpenClaw を設定**

```bash
openclaw channel add telegram
```

プロンプトが表示されたら、bot token を貼り付けます。

**ステップ 3：チャットを開始**

1. Telegram でボットを見つける（作成したユーザー名を検索）
2. `/start` を送信して開始
3. 「こんにちは！」と言ってみてください — AI アシスタントが応答するはずです！

### オプション B：Discord

Discord はコミュニティやチームに最適です。

**ステップ 1：Discord アプリケーションを作成**

1. [discord.com/developers/applications](https://discord.com/developers/applications) にアクセス
2. 「New Application」をクリックして名前を付ける
3. 「Bot」タブに移動
4. 「Add Bot」をクリックして確認
5. 「Token」の下で「Reset Token」をクリックしてコピー（安全に保存！）
6. これらの特権ゲートウェイインテントを有効化：
   - Presence Intent
   - Server Members Intent
   - Message Content Intent

**ステップ 2：OpenClaw を設定**

```bash
openclaw channel add discord
```

プロンプトが表示されたら bot token を貼り付けます。

**ステップ 3：サーバーに招待**

1. Discord Developer Portal で「OAuth2」→「URL Generator」に移動
2. スコープを選択：`bot` と `applications.commands`
3. 権限を選択：最低限、`Read Messages/View Channels`、`Send Messages`、`Read Message History`
4. 生成された URL をコピーしてブラウザで開く
5. サーバーを選択して承認

**ステップ 4：チャットを開始**

ボットがアクセスできる任意のチャネルで、メンションするか直接 DM して会話を開始してください！

### オプション C：WhatsApp

WhatsApp は QR コードペアリングが必要で、やや複雑です。

**ステップ 1：OpenClaw を設定**

```bash
openclaw channel add whatsapp
```

**ステップ 2：QR コードをスキャン**

OpenClaw はターミナルに QR コードを表示します。スマートフォンで WhatsApp を開いてスキャンします：

1. WhatsApp を開く
2. メニュー (⋮) または設定をタップ
3. 「リンク済みデバイス」をタップ
4. 「デバイスをリンク」をタップ
5. ターミナルの QR コードをスキャン

**ステップ 3：チャットを開始**

ペアリング後、リンクされたデバイス（自分自身）にメッセージを送信して接続をテストしてください！

WhatsApp のセットアップとトラブルシューティングの詳細については、[公式ドキュメント](https://openclaw.org/docs)を確認してください。

## 最初の会話

チャネルが接続されたので、AI アシスタントとの最初の会話をしましょう！

### 最初のメッセージを送信

簡単なことを試してみてください：

```
こんにちは！手伝ってもらえますか？
```

アシスタントは自己紹介をして、何が必要か尋ねるはずです。OpenClaw エージェントは、役立つ、親しみやすい、行動指向になるように設計されています。

### エージェントの応答を理解する

アシスタントに何かを依頼すると、タスクを段階的に考えていることがわかります。例えば：

```
あなた：AI に関する最新ニュースをウェブで検索して
エージェント：最近の AI ニュースを検索します。
[エージェントが web_search ツールを使用]
エージェント：見つけた内容は次のとおりです：[リンクと要約付きの結果]
```

エージェントは何をしているかを伝え、適切なツールを使用し、有用な結果を返します。

### 基本コマンド

OpenClaw はいくつかの組み込みコマンドをサポートしています：

**ステータスを確認：**
```
/status
```
現在のセッション、アクティブなモデル、システムステータスを表示します。

**ヘルプを取得：**
```
/help
```
利用可能なコマンドと基本的な使用方法を一覧表示します。

**コンテキストを表示：**
```
/context
```
現在のセッションコンテキストとメモリを表示します。

**セッションを再起動：**
```
/restart
```
現在の会話をクリアして新しく開始します。

これらのコマンドはすべてのチャネルで機能し、アシスタントを管理するための頼りになるツールです。

## 基本を理解する

### セッション

OpenClaw とのすべての会話は**セッション**で行われます。セッションを会話コンテキストと考えてください。あなたが言ったこと、進行中のタスク、関連する背景情報を記憶します。

**主要なセッション概念：**

- **分離**：各チャット（DM、チャネル、グループ）には独自のセッションがあります
- **メモリ**：セッションは会話履歴を記憶します
- **コンテキスト**：エージェントはあなたが何について話していたかを知っています
- **永続性**：セッションは再起動後も存続します（ディスクに保存されます）

エージェントが物事を忘れたり、コンテキストを混同したりする問題が発生している場合は、トラブルシューティングのヒントについて[セッションメモリガイド](/blog/troubleshooting/fix-session-memory-issues)を確認してください。

### コマンド

上記で基本的なコマンドをカバーしましたが、OpenClaw はさらに多くをサポートしています：

- `/thinking` — 推論表示を切り替え
- `/model` — AI モデルを切り替え
- `/skills` — 利用可能なスキルを一覧表示
- `/config` — 現在の設定を表示

完全なコマンドリファレンスについては、[公式ドキュメント](https://openclaw.org/docs/commands)を参照してください。

### コントロール UI

OpenClaw には、セットアップを管理するためのウェブベースのコントロールパネルが含まれています。

**UI にアクセス：**

```bash
openclaw start
```

次にブラウザで `http://localhost:3000`（デフォルトポート）を開きます。

**できること：**

- すべてのアクティブなセッションを表示
- 会話履歴を表示
- チャネル設定を管理
- システムの健全性を監視
- エージェント設定を調整
- ログとデバッグ情報を表示

コントロール UI は、複数のチャネルを実行したり、問題をデバッグしたりする場合に特に役立ちます。

## 一般的なタスク

OpenClaw で実際に**何ができるか**を探りましょう。

### コード支援

OpenClaw はプログラミングタスクの支援に優れています：

```
あなた：CSV ファイルを読み込んで、2列目の数値の平均を計算する Python スクリプトを作成して

エージェント：[コードを作成して説明]
```

エージェントはワークスペース内のファイルを読み取り、新しいコードを書き、複雑な概念を説明し、エラーをデバッグすることもできます。

### カレンダーとメール管理

適切なスキルを有効にすると：

```
あなた：明日のカレンダーには何がある？
エージェント：[カレンダーをチェックしてイベントを一覧表示]

あなた：john@example.com に会議についてメールを送って
エージェント：[メールを下書きして送信前に確認を求める]
```

### ウェブ検索と調査

エージェントはウェブを検索して発見を要約できます：

```
あなた：React hooks のベストプラクティスを調査して要約して
エージェント：[検索、記事を読み、包括的な要約を提供]
```

### ブラウザ自動化

[Browser Relay](/blog/browser-relay) 機能を使用すると、エージェントはウェブブラウザを制御できます：

```
あなた：example.com に行って、私の詳細で連絡フォームを記入して
エージェント：[ブラウザを開き、ナビゲートし、フォームを記入して送信]
```

これは、繰り返しのウェブタスクを自動化したり、ウェブアプリケーションをテストしたりするのに非常に強力です。

### スマートホーム制御

Home Assistant を統合している場合：

```
あなた：すべてのライトを消して
エージェント：[Home Assistant にコマンドを送信]

あなた：リビングルームの温度は？
エージェント：[センサーをチェックして報告]
```

### ファイル管理と整理

```
あなた：ダウンロードフォルダのすべての PDF ファイルを日付順に整理して
エージェント：[フォルダをスキャン、日付ベースのサブフォルダを作成、ファイルを移動]
```

可能性は無限です！アシスタントは強力なツールにアクセスでき、それらを創造的に組み合わせて複雑なタスクを解決できます。

## トラブルシューティング

問題が発生していますか？一般的な問題と解決策への迅速なリンクは次のとおりです：

### 認証の問題

API 認証エラーが発生している場合：
- [Claude 認証エラーの修正](/blog/troubleshooting/fix-claude-authentication-errors)

### ツールエラー

「tool_use_id」または類似のエラーが表示される場合：
- [ツール使用 ID エラーの修正](/blog/troubleshooting/fix-tool-use-id-error)

### Cron と Heartbeat の問題

スケジュールされたタスクまたはハートビートが機能していない場合：
- [Cron/Heartbeat 問題の修正](/blog/troubleshooting/fix-cron-heartbeat-not-working)

### セッションメモリの問題

エージェントが物事を忘れたり、コンテキストが混同されたりする場合：
- [セッションメモリ問題の修正](/blog/troubleshooting/fix-session-memory-issues)

### 一般的なデバッグ

一般的なトラブルシューティングの場合：
1. ログを確認：`openclaw logs`
2. ゲートウェイを再起動：`openclaw restart`
3. 設定を確認：`openclaw config show`
4. システムステータスを確認：チャットで `/status` コマンド

問題が解決しない場合は、[OpenClaw GitHub Issues](https://github.com/openclaw/openclaw/issues) にアクセスするか、コミュニティ Discord に参加してください。

目標はシンプルです：デジタルライフの退屈な部分を自動化することで、時間と精神的エネルギーを取り戻すこと。AI アシスタントはここにいて助けてくれます — 働かせましょう！
